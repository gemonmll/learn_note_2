# 例子 1：先体验 `tcmalloc` 替换默认分配器

这个例子不做 profile，只是感受“替换分配器”这件事。

### 代码：`alloc_bench.cpp`

```cpp
#include <chrono>
#include <iostream>
#include <string>
#include <vector>

int main() {
    constexpr int rounds = 200;
    constexpr int n = 200000;

    auto start = std::chrono::steady_clock::now();

    for (int r = 0; r < rounds; ++r) {
        std::vector<std::string*> v;
        v.reserve(n);

        for (int i = 0; i < n; ++i) {
            v.push_back(new std::string("hello gperftools"));
        }

        for (auto p : v) {
            delete p;
        }
    }

    auto end = std::chrono::steady_clock::now();
    auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count();

    std::cout << "elapsed = " << ms << " ms\n";
    return 0;
}
```

### 普通编译

```bash
g++ -O2 -g alloc_bench.cpp -o alloc_bench
./alloc_bench
```

### 用 `tcmalloc` 编译

```bash
g++ -O2 -g alloc_bench.cpp -o alloc_bench_tcmalloc -ltcmalloc
./alloc_bench_tcmalloc
```

### 或者不用重新编译，直接注入

官方说明也支持 `LD_PRELOAD` 方式。 ([GitHub][1])

```bash
g++ -O2 -g alloc_bench.cpp -o alloc_bench
LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libtcmalloc.so ./alloc_bench
```

### 你该观察什么

看运行时间是否下降。
这个例子里有大量小对象 `new/delete`，很适合观察分配器差异。

# 例子 2：用 CPU Profiler 找“最耗时函数”

这个例子最适合初学者，因为结果直观。

### 代码：`cpu_profile_demo.cpp`

```cpp
#include <cmath>
#include <cstdint>
#include <iostream>
#include <vector>

static bool is_prime(std::uint64_t x) {
    if (x < 2) return false;
    for (std::uint64_t i = 2; i * i <= x; ++i) {
        if (x % i == 0) return false;
    }
    return true;
}

static double heavy_compute(int n) {
    double s = 0.0;
    for (int i = 1; i <= n; ++i) {
        s += std::sqrt(i) * std::sin(i);
    }
    return s;
}

int main() {
    std::uint64_t prime_count = 0;
    double sum = 0.0;

    for (int round = 0; round < 5; ++round) {
        for (std::uint64_t i = 2; i < 200000; ++i) {
            if (is_prime(i)) {
                ++prime_count;
            }
        }
        sum += heavy_compute(5000000);
    }

    std::cout << "prime_count = " << prime_count << "\n";
    std::cout << "sum = " << sum << "\n";
    return 0;
}
```

### 编译

官方 quick start 说明，CPU profiler 需要链接 `-lprofiler`。 ([GitHub][1])

```bash
g++ -O2 -g cpu_profile_demo.cpp -o cpu_profile_demo -lprofiler -lm
```

### 运行并生成 profile

官方推荐直接设置 `CPUPROFILE`。 ([GitHub][1])

```bash
CPUPROFILE=/tmp/cpu.prof ./cpu_profile_demo
```

### 分析结果

官方说明可用 `pprof` 分析。 ([GitHub][1])

```bash
pprof ./cpu_profile_demo /tmp/cpu.prof
```

常见还能这样看：

```bash
pprof --text ./cpu_profile_demo /tmp/cpu.prof
pprof --callgrind ./cpu_profile_demo /tmp/cpu.prof > cpu.callgrind
```

### 你会看到什么

大概率热点会集中在：

* `is_prime`
* `heavy_compute`
* `sqrt`
* `sin`

这就是 CPU profiler 的核心用途：
**告诉你时间主要花在哪几个函数里。**

# 例子 3：用 Heap Profiler 看“内存从哪分配出来”

官方文档说明，heap profiler 会跟踪每个分配点，也就是分配时的活动调用栈。 ([GPerftools][2])

### 代码：`heap_profile_demo.cpp`

```cpp
#include <chrono>
#include <iostream>
#include <string>
#include <thread>
#include <vector>

static std::vector<char*> g_leaks;

void leak_some_memory(int blocks, int size) {
    for (int i = 0; i < blocks; ++i) {
        char* p = new char[size];
        p[0] = 'A';
        g_leaks.push_back(p);  // 故意不释放，方便观察
    }
}

void temporary_alloc(int blocks, int size) {
    std::vector<char*> tmp;
    for (int i = 0; i < blocks; ++i) {
        char* p = new char[size];
        p[0] = 'B';
        tmp.push_back(p);
    }
    for (auto p : tmp) {
        delete[] p;
    }
}

int main() {
    for (int round = 0; round < 5; ++round) {
        temporary_alloc(1000, 1024);   // 约 1 MB，临时申请后释放
        leak_some_memory(500, 4096);   // 约 2 MB，故意保留
        std::this_thread::sleep_for(std::chrono::seconds(1));
        std::cout << "round " << round << " done\n";
    }

    std::cout << "finish\n";
    return 0;
}
```

### 编译

官方 quick start 写的是：heap profiler 常见做法是链接 `-ltcmalloc`。 ([GitHub][1])

```bash
g++ -O2 -g heap_profile_demo.cpp -o heap_profile_demo -ltcmalloc -lpthread
```

### 运行

设置 `HEAPPROFILE`。官方会按前缀生成多个 `.heap` 文件。 ([GitHub][1])

```bash
HEAPPROFILE=/tmp/heapdemo ./heap_profile_demo
```

运行后你通常会看到类似文件：

```bash
ls /tmp/heapdemo*.heap
```

### 分析

```bash
pprof ./heap_profile_demo /tmp/heapdemo.0001.heap
pprof --text ./heap_profile_demo /tmp/heapdemo.0005.heap
```

### 你该怎么理解结果

你会看到哪些函数分配了最多内存。
这个例子里通常会看到：

* `leak_some_memory`
* `temporary_alloc`

其中 `leak_some_memory` 的“活跃内存”会持续增长，`temporary_alloc` 则更像“短时申请后释放”。

# 例子 4：代码里手动控制 CPU profiler

除了环境变量方式，`gperftools` 也支持你在代码里手动开启/停止 profiler。仓库文档提到 `PROFILESELECTED=1` 和 `ProfilerEnable()/ProfilerDisable()` 这类选择性采样控制。 ([GitHub][3])

这个方式适合你只想分析某一段代码，而不是整个程序。

### 代码：`manual_cpu_profile.cpp`

```cpp
#include <gperftools/profiler.h>
#include <cmath>
#include <iostream>

double hot_function() {
    double s = 0.0;
    for (int i = 1; i < 10000000; ++i) {
        s += std::sqrt(i) * std::cos(i);
    }
    return s;
}

double cold_function() {
    double s = 0.0;
    for (int i = 1; i < 100000; ++i) {
        s += std::sqrt(i);
    }
    return s;
}

int main() {
    cold_function();

    ProfilerStart("/tmp/manual.prof");
    double x = hot_function();
    ProfilerStop();

    cold_function();

    std::cout << x << "\n";
    return 0;
}
```

### 编译

```bash
g++ -O2 -g manual_cpu_profile.cpp -o manual_cpu_profile -lprofiler -lm
```

### 运行

```bash
./manual_cpu_profile
pprof --text ./manual_cpu_profile /tmp/manual.prof
```

### 这个例子的意义

你会发现 profile 主要集中在 `hot_function()`，
因为我们只对那一段进行了采样。

# 一个更接近实际项目的小套路

## 场景 A：程序感觉“慢”

先做 CPU profiling：

```bash
g++ -O2 -g app.cpp -o app -lprofiler
CPUPROFILE=/tmp/app.cpu ./app
pprof --text ./app /tmp/app.cpu
```

然后你看前几名热点函数，优先优化它们。

---

## 场景 B：程序内存越来越大

先做 heap profiling：

```bash
g++ -O2 -g app.cpp -o app -ltcmalloc
HEAPPROFILE=/tmp/app.heap ./app
pprof --text ./app /tmp/app.heap.0005.heap
```

然后重点看：

* 哪个函数分配最多
* 哪些对象一直活着
* 哪条调用链在增长

---

## 场景 C：只是想让 `new/malloc` 更快

只换分配器：

```bash
g++ -O2 -g app.cpp -o app -ltcmalloc
```

或者：

```bash
LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libtcmalloc.so ./app
```

---

# 学习时你要注意的几个点

### 1. 一定加 `-g`

不然分析结果里函数名、调用栈会难看很多。

### 2. 最好先别开太强优化

学习阶段可以试 `-O0 -g` 或 `-O1 -g`，更容易看明白调用关系。
真正测性能再用 `-O2 -g`。

### 3. Heap checker 现在不建议当主力

仓库说明里提到 **heap checker 从 gperftools 2.11 起已弃用，建议考虑 ASan 等 sanitizer**。 ([GitHub][4])

### 4. profiler 通常是“运行时打开”

仓库文档说明，这些 profiler 在没有启用时会保持 dormant，不会一直消耗资源；常见通过环境变量开启。 ([GitHub][3])

---

# 你可以按这个顺序练习

最推荐学习路径：

1. 先跑 `例子 2`，学会看 CPU 热点
2. 再跑 `例子 3`，学会看内存分配来源
3. 最后跑 `例子 1`，体会 `tcmalloc` 对分配性能的影响
