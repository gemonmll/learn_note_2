```cpp
  /**
   * @brief Get the Reader object that subscribe `channel_name`
   *
   * @tparam MessageT Message Type
   * @param channel_name channel name
   * @return std::shared_ptr<Reader<MessageT>> result reader
   */
  template <typename MessageT>
  auto GetReader(const std::string& channel_name)
      -> std::shared_ptr<Reader<MessageT>>; 
```

这里的 `->` 不是“指针访问成员”的那个箭头，
而是 **函数返回类型后置写法**，叫 **trailing return type**。

你这句：

```cpp id="2k06db"
template <typename MessageT>
auto GetReader(const std::string& channel_name)
    -> std::shared_ptr<Reader<MessageT>>;
```

等价于传统写法：

```cpp id="b3o755"
template <typename MessageT>
std::shared_ptr<Reader<MessageT>> GetReader(const std::string& channel_name);
```

---

## 它怎么读

可以读成：

> `GetReader` 这个函数，参数是 `const std::string& channel_name`，返回值类型是 `std::shared_ptr<Reader<MessageT>>`

只是写法换成了：

* 前面先写 `auto`
* 参数列表写完后
* 用 `-> 返回类型`

---

# 为什么要这么写

最直接的原因是：

**有些函数返回类型里要用到参数名，或者写在前面很难表达，于是 C++11 提供了这种后置返回类型。**

---

## 普通例子

### 传统写法

```cpp id="2qo5tg"
int add(int a, int b);
```

### 后置返回类型写法

```cpp id="ri7fgn"
auto add(int a, int b) -> int;
```

两者一样。

---

# 你这段里为什么作者可能喜欢这么写

因为模板函数里返回类型经常很长，比如：

```cpp id="qlbrmu"
std::shared_ptr<Reader<MessageT>>
```

写成后置以后，函数名会更突出一点：

```cpp id="epm9h2"
auto GetReader(...) -> std::shared_ptr<Reader<MessageT>>
```

可读性有时更好。

---

# 它和“自动推导返回值”的 `auto` 不是一回事

这个很容易混。

## 这种写法

```cpp id="6it2i9"
auto GetReader(...) -> std::shared_ptr<Reader<MessageT>>;
```

这里的返回类型**不是自动推导**，
真正的返回类型已经明确写在 `->` 后面了。

---

## 而这种才是自动推导

```cpp id="rg3o01"
auto add(int a, int b) {
    return a + b;
}
```

这里编译器根据 `return a + b;` 推导返回 `int`。

---

# 为什么不用直接写前面那种

你这句完全可以写成：

```cpp id="ylr1p2"
template <typename MessageT>
std::shared_ptr<Reader<MessageT>> GetReader(const std::string& channel_name);
```

语义没区别。

所以这里主要是**风格选择**。

---

# 后置返回类型最有价值的场景

虽然你这个例子里只是风格问题，但它真正厉害的地方是在返回类型依赖参数时。

比如：

```cpp id="dhkkg4"
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b);
```

这里如果写成传统形式就麻烦，因为前面还没出现 `a` 和 `b`：

```cpp id="ktg1j1"
decltype(a + b) add(T a, U b);   // 不行，a b 还没声明
```

所以后置返回类型可以让你在返回类型里用参数名。

---

# 你这个例子逐段拆开看

```cpp id="onb9am"
template <typename MessageT>
```

这是模板函数，模板参数是 `MessageT`

```cpp id="x41yxz"
auto GetReader(const std::string& channel_name)
```

函数名是 `GetReader`，参数是 `channel_name`

```cpp id="vf8prc"
-> std::shared_ptr<Reader<MessageT>>;
```

返回类型是 `std::shared_ptr<Reader<MessageT>>`

---

# 再强调一下：这里的 `->` 不是这两个意思

## 不是成员访问

不是这种：

```cpp id="6q8mnc"
ptr->member
```

## 也不是 lambda 里必然的那个

虽然 lambda 也常用：

```cpp id="mz2ssb"
[]() -> int { return 1; }
```

但本质上还是同一个概念：**后置返回类型**。

---

# 一句话记忆

你这句里的 `->` 就是：

**“函数参数写完后，再写返回类型”**

所以：

```cpp id="xlhmb1"
auto GetReader(...) -> std::shared_ptr<Reader<MessageT>>
```

等价于：

```cpp id="e8j2kw"
std::shared_ptr<Reader<MessageT>> GetReader(...)
```

---

你要的话，我下一条可以顺着给你讲：
**为什么模板代码里很爱用 `auto ... -> ...`，以及它和 `decltype` 怎么配合。**
