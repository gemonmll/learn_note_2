# 先看结论

## 什么时候要加 `typename`

当你写了一个**依赖于模板参数的嵌套名字**，并且这个名字表示的是**类型**时，通常要加：

```cpp id="xvg1ny"
typename SomeTemplate<T>::TypeName
```

---

## 什么时候不用

如果这个名字**不依赖模板参数**，或者编译器已经明确知道它是类型，就不用。

例如：

```cpp id="z2hnwd"
std::vector<int>::value_type x;
```

这里不用 `typename`，因为 `std::vector<int>` 已经是具体类型了。

---

# 一、先拆你这句代码

你这里有：

```cpp id="mbsn5t"
typename Client<Request, Response>::SharedResponse
```

拆开看：

* `Client<Request, Response>`：是一个模板实例
* `::SharedResponse`：是这个类里面定义的一个名字
* 这个名字来自类里：

```cpp id="x2w1bf"
using SharedResponse = typename std::shared_ptr<Response>;
```

所以 `SharedResponse` 的实际含义是：

```cpp id="v8fj0l"
std::shared_ptr<Response>
```

---

## 那为什么还要写 `typename`

因为在函数定义这一行，编译器看到：

```cpp id="4f57da"
Client<Request, Response>::SharedResponse
```

时，它第一眼只知道：

* `Request` 和 `Response` 都还是模板参数
* 所以 `Client<Request, Response>` 的具体类型还没最终确定
* 那 `SharedResponse` 到底是不是一个类型名，它现在不能百分百确定

因此你要写：

```cpp id="f5e63m"
typename Client<Request, Response>::SharedResponse
```

明确告诉编译器：

> `SharedResponse` 是个类型，不是静态成员、枚举值或者别的东西。

---

# 二、为什么编译器会不知道

因为 C++ 里像这种：

```cpp id="b67743"
A::B
```

`B` 可能是很多东西：

* 类型
* 静态成员变量
* 枚举值
* 静态函数

在模板里如果 `A` 还依赖模板参数，编译器第一次看代码时就不敢猜。

所以：

```cpp id="dkpw1l"
Client<Request, Response>::SharedResponse
```

它会想：

* `SharedResponse` 是类型？
* 还是别的成员？

你得用 `typename` 消除歧义。

---

# 三、你这个例子为什么是“依赖名”

因为这里：

```cpp id="ndj893"
Client<Request, Response>::SharedResponse
```

依赖于模板参数：

* `Request`
* `Response`

如果模板参数换了，`Client<Request, Response>` 对应的具体类实例也跟着变。

所以这就是**依赖名**。

只要是这种“依赖模板参数的嵌套名字”，又要把它当类型用，通常就得写 `typename`。

# 六、什么时候不用 `typename`

## 情况 1：不是依赖名

```cpp id="4copjj"
std::shared_ptr<int> p;
std::vector<int>::value_type x;
```

这些都是具体类型，不依赖模板参数，不用。

---

## 情况 2：在类模板内部声明某些成员时，当前作用域已知

比如类模板内部直接写：

```cpp id="w3swh0"
template <typename T>
class A {
    using ValueType = typename T::value_type;
};
```

这里 `T::value_type` 仍然要写 `typename`，因为它还是依赖类型名。

但如果你在类内部直接用自己已经定义好的别名，有时就不用反复写：

```cpp id="zu0seu"
template <typename T>
class A {
public:
    using ValueType = typename T::value_type;
    ValueType x;
};
```

这里 `ValueType` 已经是当前作用域里的已知类型别名，就不需要再写 `typename ValueType`。

---

## 情况 3：基类列表、构造函数初始化列表等某些特殊语法位置

C++ 有少数上下文里不允许或不需要 `typename`，但你现在先不用记太细。

你先抓主规则就够了。

# 七、你代码里另一个小点

你类里写的是：

```cpp id="4ya7q0"

using SharedResponse = typename std::shared_ptr<Response>;
```

这里前面的 `typename` 其实**不需要**。

因为：


里的 `std::shared_ptr` 已经是具体模板名，`std::shared_ptr<Request>` 明确就是一个类型，不存在歧义。

这里加 `typename` 多半是作者写多了顺手加的，或者代码风格比较保守。
