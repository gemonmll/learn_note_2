可以。你要学这个，最重要的是先抓住两个前提：

1. **锁和条件变量对象本身，必须放在共享内存里**
2. **它们必须设置成 `PTHREAD_PROCESS_SHARED`**

否则它们就只是“进程内线程同步”用的，跨进程不成立。

---

# 一、先看完整示例

我给你一个最小可运行模型：

* `writer` 进程：写入数据并唤醒
* `reader` 进程：阻塞等待数据到来
* 共享内存里放：

  * `pthread_mutex_t`
  * `pthread_cond_t`
  * 一个 `ready` 标志
  * 一个 `value`

这里用 **POSIX 共享内存 + pthread 跨进程同步**。

---

## 1）公共头文件：`shared.h`

```cpp
#ifndef SHARED_H
#define SHARED_H

#include <pthread.h>

struct SharedData {
    pthread_mutex_t mutex;
    pthread_cond_t cond;
    int ready;
    int value;
};

constexpr const char* kShmName = "/my_process_shared_mem_demo";

#endif
```

---

## 2）初始化进程：`init.cpp`

这个程序负责：

* 创建共享内存
* 初始化 mutex 和 cond 的属性
* 设置为跨进程共享
* 初始化共享数据

```cpp
#include "shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cerrno>
#include <cstdio>
#include <cstdlib>
#include <cstring>

int main() {
    int fd = shm_open(kShmName, O_CREAT | O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    if (ftruncate(fd, sizeof(SharedData)) == -1) {
        perror("ftruncate");
        close(fd);
        shm_unlink(kShmName);
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedData),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        shm_unlink(kShmName);
        return 1;
    }

    auto* data = static_cast<SharedData*>(addr);

    pthread_mutexattr_t mutex_attr;
    pthread_condattr_t cond_attr;

    if (pthread_mutexattr_init(&mutex_attr) != 0) {
        fprintf(stderr, "pthread_mutexattr_init failed\n");
        return 1;
    }
    if (pthread_condattr_init(&cond_attr) != 0) {
        fprintf(stderr, "pthread_condattr_init failed\n");
        return 1;
    }

    if (pthread_mutexattr_setpshared(&mutex_attr, PTHREAD_PROCESS_SHARED) != 0) {
        fprintf(stderr, "pthread_mutexattr_setpshared failed\n");
        return 1;
    }
    if (pthread_condattr_setpshared(&cond_attr, PTHREAD_PROCESS_SHARED) != 0) {
        fprintf(stderr, "pthread_condattr_setpshared failed\n");
        return 1;
    }

    if (pthread_mutex_init(&data->mutex, &mutex_attr) != 0) {
        fprintf(stderr, "pthread_mutex_init failed\n");
        return 1;
    }
    if (pthread_cond_init(&data->cond, &cond_attr) != 0) {
        fprintf(stderr, "pthread_cond_init failed\n");
        return 1;
    }

    data->ready = 0;
    data->value = 0;

    pthread_mutexattr_destroy(&mutex_attr);
    pthread_condattr_destroy(&cond_attr);

    printf("init done\n");

    munmap(addr, sizeof(SharedData));
    close(fd);
    return 0;
}
```

## 3）等待进程：`reader.cpp`

这个进程阻塞等待，直到 `ready == 1`。

```cpp
#include "shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kShmName, O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedData),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    auto* data = static_cast<SharedData*>(addr);

    pthread_mutex_lock(&data->mutex);

    while (!data->ready) {
        printf("reader: waiting...\n");
        pthread_cond_wait(&data->cond, &data->mutex);
    }

    printf("reader: got value = %d\n", data->value);

    pthread_mutex_unlock(&data->mutex);

    munmap(addr, sizeof(SharedData));
    close(fd);
    return 0;
}
```

## 4）写入并唤醒进程：`writer.cpp`

```cpp
#include "shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kShmName, O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedData),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    auto* data = static_cast<SharedData*>(addr);

    pthread_mutex_lock(&data->mutex);

    data->value = 42;
    data->ready = 1;

    printf("writer: signal reader\n");
    pthread_cond_signal(&data->cond);

    pthread_mutex_unlock(&data->mutex);

    munmap(addr, sizeof(SharedData));
    close(fd);
    return 0;
}
```

## 5）清理程序：`cleanup.cpp`

```cpp
#include "shared.h"

#include <fcntl.h>
```

# 四、这个例子里最关键的点

---

## 1. 为什么锁和条件变量必须放在共享内存里

因为它们本身不是“魔法”，而是有内部状态的数据结构。

比如 mutex 内部至少要记录：

* 是否已加锁
* 谁在等待
* 状态字

cond 内部也要记录等待队列相关状态。

如果两个进程各自有一份 `pthread_mutex_t` 副本，那它们锁的根本不是同一个对象。

所以必须把它们和共享数据一起放在共享内存里：

```cpp
struct SharedData {
    pthread_mutex_t mutex;
    pthread_cond_t cond;
    int ready;
    int value;
};
```

---

## 2. 为什么一定要 `PTHREAD_PROCESS_SHARED`

默认情况下：

* `pthread_mutex_t`
* `pthread_cond_t`

都是**进程私有**的，也就是只给同一个进程里的线程用。

要跨进程，必须显式设置：

```cpp
pthread_mutexattr_setpshared(&mutex_attr, PTHREAD_PROCESS_SHARED);
pthread_condattr_setpshared(&cond_attr, PTHREAD_PROCESS_SHARED);
```

没有这两句，这个例子通常就不成立。

---

## 3. 为什么 `pthread_cond_wait` 前面要写 `while`

一定要这样：

```cpp
while (!data->ready) {
    pthread_cond_wait(&data->cond, &data->mutex);
}
```

不要写成：

```cpp
if (!data->ready) {
    pthread_cond_wait(&data->cond, &data->mutex);
}
```

因为条件变量的正确用法一直都是“**条件 + while**”。

原因有两个：

1. 可能有**伪唤醒**
2. 被唤醒不等于条件一定满足，也可能是别的等待者抢先改变了状态

所以条件变量只负责“通知你再检查一次条件”，不保证条件一定成立。

---

## 4. `pthread_cond_wait` 为什么要带 mutex

因为它做的是一个原子操作：

1. 先把当前线程/进程加入等待队列
2. 释放 mutex
3. 进入睡眠
4. 被唤醒后重新抢 mutex
5. 返回给你

这能避免“检查完条件，还没睡着，对方已经 signal 了，结果信号丢了”的竞态问题。

所以条件变量永远不是单独用，必须和互斥锁配套。

# 五、再给你一个更贴近思维模型的时序

## reader

```text
lock(mutex)
while ready == 0:
    cond_wait(cond, mutex)
读取 value
unlock(mutex)
```

## writer

```text
lock(mutex)
写 value
ready = 1
signal(cond)
unlock(mutex)
```

这就是最标准的生产者-消费者里“一次通知”的基本形态。

# 六、常见错误

---

## 错误 1：把 mutex/cond 放在普通全局变量里

比如：

```cpp
pthread_mutex_t g_mutex;
pthread_cond_t g_cond;
```

然后两个独立进程都各自运行，觉得能同步。

这是错的。

因为这是两个进程各自地址空间里的两份不同对象。

---

## 错误 2：用了共享内存，但没设 `PTHREAD_PROCESS_SHARED`

这样也不对。

对象虽然在共享内存，但 pthread 实现仍按“进程内专用”处理，跨进程行为不可靠。

---

## 错误 3：不用 while 检查条件

这个会埋下非常隐蔽的 bug。

---

## 错误 4：writer 不加锁就改 `ready/value`

如果共享状态有多个字段，必须受同一个 mutex 保护，不然 reader 可能看到不一致状态。

---

# 七、一个稍微进阶一点的版本：多次通信

上面是“一次 ready 后退出”。

如果你想循环生产/消费，可以这样改：

reader:

```cpp
for (;;) {
    pthread_mutex_lock(&data->mutex);
    while (!data->ready) {
        pthread_cond_wait(&data->cond, &data->mutex);
    }

    int v = data->value;
    data->ready = 0;

    pthread_mutex_unlock(&data->mutex);

    printf("reader got %d\n", v);
}
```

writer:

```cpp
for (int i = 1; i <= 10; ++i) {
    pthread_mutex_lock(&data->mutex);

    while (data->ready) {
        pthread_mutex_unlock(&data->mutex);
        usleep(1000);
        pthread_mutex_lock(&data->mutex);
    }

    data->value = i;
    data->ready = 1;
    pthread_cond_signal(&data->cond);

    pthread_mutex_unlock(&data->mutex);
}
```

不过这版还比较粗糙。更正规的做法一般会再加一个“消费者条件变量”或者用信号量做双向同步。

# 八、什么时候用跨进程 mutex + cond，什么时候不用

适合：

* 两三个相关进程共享状态
* 本机内通信
* 低延迟
* 想要阻塞等待而不是轮询

不太适合：

* 跨机器
* 需要复杂路由
* 需要持久化消息
* 要兼容很多不同语言和环境

那种场景更像 socket、消息队列、RPC。

---

# 九、你现在最应该形成的理解

跨进程锁和条件变量并不神秘，本质上就是：

* 把“锁对象”和“条件变量对象”也当成共享数据
* 一起放进共享内存
* 告诉 pthread：它们是 `PTHREAD_PROCESS_SHARED`
* 然后按线程里的经典写法使用

也就是：

**跨进程版线程同步 = 共享内存 + 进程共享属性 + 正确的 mutex/cond 用法**

---

# 十、一句话总结

**跨进程锁和跨进程条件变量是可以用的，但必须把 `pthread_mutex_t` 和 `pthread_cond_t` 放在共享内存中，并设置为 `PTHREAD_PROCESS_SHARED`；之后它们的用法和线程内的“mutex + cond + while 条件检查”基本一样。**
