```cpp
template <typename M0, typename M1 = NullType, typename M2 = NullType,
          typename M3 = NullType>
class DataFusion {
 public:
  virtual ~DataFusion() {}
  virtual bool Fusion(uint64_t* index, std::shared_ptr<M0>& m0,  // NOLINT
                      std::shared_ptr<M1>& m1,                   // NOLINT
                      std::shared_ptr<M2>& m2,                   // NOLINT
                      std::shared_ptr<M3>& m3) = 0;              // NOLINT
};

template <typename M0, typename M1, typename M2>
class DataFusion<M0, M1, M2, NullType> {
 public:
  virtual ~DataFusion() {}

  virtual bool Fusion(uint64_t* index, std::shared_ptr<M0>& m0,  // NOLINT
                      std::shared_ptr<M1>& m1,                   // NOLINT
                      std::shared_ptr<M2>& m2) = 0;              // NOLINT
};

template <typename M0, typename M1>
class DataFusion<M0, M1, NullType, NullType> {
 public:
  virtual ~DataFusion() {}

  virtual bool Fusion(uint64_t* index, std::shared_ptr<M0>& m0,  // NOLINT
                      std::shared_ptr<M1>& m1) = 0;              // NOLINT
}; 已有了nulltype,为什么还有template <typename M0, typename M1, typename M2>

```


这里的 `NullType` 通常就是一个**占位类型**，意思是：

“这个模板参数位先留着，但当前其实没有这个消息类型。”

所以这段代码的设计意图是让 `DataFusion` 最多支持 4 路输入：

```cpp
template <typename M0, typename M1 = NullType, typename M2 = NullType,
          typename M3 = NullType>
class DataFusion
```

其中：

* `M0` 是必填
* `M1 / M2 / M3` 是可选
* 如果你只需要 1 路数据融合，那么后面 3 个就默认是 `NullType`

比如：

```cpp
DataFusion<Image>
```

实际等价于：

```cpp
DataFusion<Image, NullType, NullType, NullType>
```

---

## 它一般是什么

`NullType` 往往是框架里自己定义的一个空类型，类似：

```cpp
struct NullType {};
```

或者：

```cpp
class NullType {};
```

它本身通常没有成员，没有实际业务含义，只是为了：

1. **补齐模板参数个数**
2. **表示“没有这个类型”**
3. **方便写统一接口**

---

## 为什么不用 `void`

因为这里函数参数写的是：

```cpp
std::shared_ptr<M1>& m1
```

如果 `M1 = void`，那就会变成：

```cpp
std::shared_ptr<void>& m1
```

虽然技术上有时能存在，但语义不够清晰，而且后续特化、类型判断、重载处理都不如 `NullType` 方便。

`NullType` 的好处是：

* 类型明确
* 可做模板特化
* 可用 `std::is_same<M1, NullType>` 判断是否启用某一路

---

## 在这段代码里它的作用

这套接口想统一 1~4 路融合逻辑，所以写成固定形式：

```cpp
Fusion(index, m0, m1, m2, m3)
```

但不是每个融合器都真有 4 路输入。
于是不用的输入类型就拿 `NullType` 顶上。

例如二路融合可能是：

```cpp
class MyFusion : public DataFusion<LidarMsg, CameraMsg> {
 public:
  bool Fusion(uint64_t* index,
              std::shared_ptr<LidarMsg>& lidar,
              std::shared_ptr<CameraMsg>& camera,
              std::shared_ptr<NullType>&,
              std::shared_ptr<NullType>&) override {
    ...
  }
};
```

意思就是后两路根本不用。

---

## 你可以把它理解成

`NullType` ≈ 模板世界里的“无 / 空参数占位符”。

它不是空指针，也不是 `nullptr`，而是一个**类型层面的占坑对象**。

## 怎么确认你这个工程里的 `NullType`

直接全局搜：

```bash
grep -R "class NullType\|struct NullType" .
```

或者：

```bash
grep -R "NullType" /apollo | head -n 20
```

大概率能看到定义位置。

这个 `NullType` 不是特殊语法，就是项目里定义的一个“空类型占位符”，用来表示 `M1/M2/M3` 这些模板参数当前未使用。
