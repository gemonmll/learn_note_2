`thread_local` 是 C++ 里的**线程局部存储**关键字。

它的作用是：

> **同一个变量名，在每个线程里各有一份独立副本，互不影响。**

---

# 1. 先看最直观的例子

```cpp
#include <iostream>
#include <thread>

thread_local int counter = 0;

void work(const char* name) {
    ++counter;
    ++counter;
    std::cout << name << ": " << counter << std::endl;
}

int main() {
    std::thread t1(work, "t1");
    std::thread t2(work, "t2");

    t1.join();
    t2.join();

    work("main");
    return 0;
}
```

可能输出：

```cpp
t1: 2
t2: 2
main: 2
```

这里虽然都是同一个 `counter`，但：

* `t1` 有自己的 `counter`
* `t2` 有自己的 `counter`
* 主线程也有自己的 `counter`

彼此不共享。

---

# 2. 如果不用 `thread_local` 会怎样

如果写成：

```cpp
int counter = 0;
```

那么所有线程共享这一份变量：

* 会互相影响
* 还可能有数据竞争
* 需要加锁保护

而 `thread_local` 的意思就是：

> 不共享，每个线程自己保存一份

所以很多时候可以避免加锁。

---

# 3. 基本语法

## 修饰全局变量

```cpp
thread_local int g_value = 10;
```

---

## 修饰静态成员变量

```cpp
class A {
public:
    static thread_local int value;
};

thread_local int A::value = 0;
```

---

## 修饰函数内静态变量

```cpp
void foo() {
    thread_local int x = 0;
    ++x;
}
```

注意这里是：

* `static` 风格的生命周期
* 但每个线程独立一份

---

# 4. 它和普通局部变量有什么区别

普通局部变量：

```cpp
void foo() {
    int x = 0;
}
```

特点：

* 每次调用函数都会重新创建
* 函数结束就销毁

---

`thread_local` 局部静态变量：

```cpp
void foo() {
    thread_local int x = 0;
}
```

特点：

* 对每个线程，只初始化一次
* 在线程整个生命周期内都存在
* 下次这个线程再调用 `foo()`，还是那一份

---

# 5. 一个更容易看懂的例子

```cpp
#include <iostream>
#include <thread>

void test() {
    thread_local int x = 0;
    ++x;
    std::cout << std::this_thread::get_id() << " x = " << x << std::endl;
}

int main() {
    std::thread t1([] {
        test();
        test();
    });

    std::thread t2([] {
        test();
        test();
    });

    t1.join();
    t2.join();
}
```

效果类似：

* `t1` 第一次调用 `test()`，`x=1`
* `t1` 第二次调用 `test()`，`x=2`
* `t2` 也有自己的 `x`，从 1 开始

所以每个线程都像有一个私有缓存。

---

# 6. 常见用途

## 1）保存线程自己的上下文

比如日志 trace id、线程编号：

```cpp
thread_local std::string current_trace_id;
```

---

## 2）线程私有缓存

避免重复创建对象：

```cpp
thread_local std::vector<int> buffer;
```

每个线程复用自己的 buffer，减少分配开销。

---

## 3）线程私有随机数引擎

```cpp
thread_local std::mt19937 rng(std::random_device{}());
```

这样每个线程有自己的随机数生成器，不用加锁。

---

## 4）协程/调度器里记录“当前线程运行环境”

例如：

```cpp
thread_local Scheduler* current_scheduler = nullptr;
thread_local CRoutine* current_routine = nullptr;
```

这在你前面看的 Cyber/协程代码里很常见。

---

# 7. 初始化时机

`thread_local` 变量的初始化是：

> **每个线程第一次使用到这个变量时，初始化一次。**

例如：

```cpp
thread_local int x = Init();
```

那么：

* 线程 A 第一次访问 `x`，调用一次 `Init()`
* 线程 B 第一次访问 `x`，再调用一次 `Init()`

不是全局只初始化一次。

---

# 8. 生命周期

`thread_local` 变量的生命周期是：

> 从该线程初始化它开始，到该线程结束为止。

也就是说：

* 线程活着，它活着
* 线程退出，它析构

如果是对象类型：

```cpp
thread_local std::string name = "worker";
```

在线程结束时会调用析构函数。

---

# 9. 和 `static` 的区别

很多人会混。

## `static`

整个进程共享一份：

```cpp
static int x = 0;
```

所有线程都访问同一个 `x`。

---

## `thread_local`

每个线程一份：

```cpp
thread_local int x = 0;
```

每个线程独立。

---

## `static thread_local`

在类静态成员或函数静态局部里常见，本质重点还是 `thread_local`：

```cpp
static thread_local int x = 0;
```

表示：

* 静态存储期
* 线程局部副本

---

# 10. 一个很典型的对比例子

```cpp
#include <iostream>
#include <thread>
#include <mutex>

int shared_num = 0;
thread_local int local_num = 0;
std::mutex mtx;

void work() {
    for (int i = 0; i < 3; ++i) {
        ++local_num;

        std::lock_guard<std::mutex> lock(mtx);
        ++shared_num;
        std::cout << "thread " << std::this_thread::get_id()
                  << " local=" << local_num
                  << " shared=" << shared_num << std::endl;
    }
}
```

这里：

* `shared_num` 是全线程共享
* `local_num` 是每个线程独享

---

# 11. 使用时的注意点

## 1）它不能解决“线程间共享”问题

如果你本来就是想让多个线程看到同一个值，那就不该用 `thread_local`。

---

## 2）它只能在线程内部看到自己的副本

别指望线程 A 改完后线程 B 能读到。

---

## 3）对象太大时要小心

如果你写：

```cpp
thread_local BigObject obj;
```

那每个线程都会有一份 `BigObject`，线程多了会很占内存。

---

## 4）析构时机要注意

线程退出时才析构，有些线程池线程长期不退出，那么这些对象也会一直占着资源。

---

# 12. 在工程里常见的写法

## 当前线程 ID 缓存

```cpp
thread_local int t_cached_tid = 0;
```

---

## 当前线程名字

```cpp
thread_local const char* t_thread_name = "unknown";
```

---

## 当前协程指针

```cpp
thread_local CRoutine* current_croutine = nullptr;
```

这类写法在高性能框架里特别常见。

---

# 13. 面试式回答可以怎么说

你可以这么答：

> `thread_local` 用来声明线程局部变量。
> 它的特点是每个线程都有自己独立的一份实例，互不共享，所以访问时通常不需要加锁。
> 它适合保存线程上下文、线程私有缓存、随机数引擎、当前调度器/协程指针等信息。
> 生命周期通常持续到线程结束。

---

# 14. 一句话总结

**`thread_local` = 每个线程各自独立的一份“全局/静态变量”。**
