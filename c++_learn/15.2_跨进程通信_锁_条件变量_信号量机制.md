能，**但不是默认就能跨进程**。

你这个疑问非常关键：
很多人只学过“线程里的 mutex / condition_variable”，于是会以为这些同步原语天然只属于同一进程。其实在 Linux/POSIX 下，**有一部分同步原语可以做成“进程共享”**。

## 先给结论

### 可以跨进程的

* **POSIX mutex**：可以，但要设置成 `PTHREAD_PROCESS_SHARED`
* **POSIX condition variable**：可以，但也要设置成 `PTHREAD_PROCESS_SHARED`
* **POSIX semaphore**

  * `sem_init(pshared != 0)` 的匿名信号量：可以
  * `sem_open` 的命名信号量：也可以
* **System V semaphore**：可以
* **futex**：可以
* **file lock / flock**：可以
* **eventfd / pipe / socket**：本来就是进程间用的

### 默认只在进程内的

* 普通默认初始化的 `pthread_mutex_t`
* 普通默认初始化的 `pthread_cond_t`
* C++ 标准库 `std::mutex`
* C++ 标准库 `std::condition_variable`

后面这几个，通常不能直接拿来做跨进程同步。

---

# 1. 为什么你会觉得条件变量只能跨线程

因为最常见的写法是：

```cpp
std::mutex m;
std::condition_variable cv;
```

这套是 **C++ 线程库对象**，默认设计目标就是：

* 同一进程内
* 多线程同步

它们背后通常基于 pthread/futex，但 **标准并没有把它们定义成跨进程同步工具**。

所以：

* `std::condition_variable`：一般只当线程间用
* `std::mutex`：一般也只当线程间用

这和 **POSIX pthread 条件变量/互斥锁** 是两层东西。

---

# 2. pthread 条件变量和锁，确实可以跨进程

重点在于：
**对象本身必须放在共享内存里，并且属性要设为进程共享。**

也就是你不能这样：

```cpp
pthread_mutex_t m;
pthread_cond_t cv;
```

放在各自进程的普通全局区/堆里，然后指望它们同步。
因为两个进程看到的根本不是同一个对象实例。

必须这样理解：

* 锁对象本身也是“共享数据”
* 条件变量对象本身也是“共享数据”

所以它们要放进共享内存，例如：

```cpp
struct SharedBlock {
    pthread_mutex_t mutex;
    pthread_cond_t cond;
    int ready;
    int value;
};
```

然后整个 `SharedBlock` 放到 `shmget/shmat` 或 `shm_open/mmap` 得到的共享内存里。

---

# 3. 仅仅放共享内存里还不够，还要设置属性

## mutex 需要：

```cpp
pthread_mutexattr_t mattr;
pthread_mutexattr_init(&mattr);
pthread_mutexattr_setpshared(&mattr, PTHREAD_PROCESS_SHARED);
pthread_mutex_init(&blk->mutex, &mattr);
```

## condvar 需要：

```cpp
pthread_condattr_t cattr;
pthread_condattr_init(&cattr);
pthread_condattr_setpshared(&cattr, PTHREAD_PROCESS_SHARED);
pthread_cond_init(&blk->cond, &cattr);
```

这个 `PTHREAD_PROCESS_SHARED` 的意思就是：

> 这个同步原语不是只给本进程线程用，而是允许多个进程共享。

如果不设，默认通常是：

```cpp
PTHREAD_PROCESS_PRIVATE
```

那就只能进程内线程同步。

---

# 4. 所以“条件变量能不能跨进程”的准确回答

**能。**

但前提是：

1. 用的是 **pthread_cond_t**，不是 `std::condition_variable`
2. 条件变量对象放在**共享内存**
3. 通过 `pthread_condattr_setpshared(..., PTHREAD_PROCESS_SHARED)` 设置成进程共享
4. 搭配的 mutex 也要是共享内存里的，并设成 `PTHREAD_PROCESS_SHARED`

---

# 5. 锁能不能跨进程

## pthread_mutex_t：能

前提同上：

* 放在共享内存
* 设置 `PTHREAD_PROCESS_SHARED`

否则不行。

---

## std::mutex：一般不要拿来做跨进程

因为标准库没有把它定义为“进程共享同步原语”。

你即使硬把某个 `std::mutex` 放共享内存，通常也不应依赖它在不同进程间正确工作。工程上这是不靠谱的。

---

# 6. 信号量更适合跨进程，理解起来也直观

信号量本来就很常用于进程间同步。

## 两类常见 POSIX 信号量

### 匿名信号量

对象在共享内存里：

```cpp
sem_t sem;
sem_init(&sem, 1, 0);
```

这里第二个参数 `pshared=1`，表示进程共享。

如果 `pshared=0`，那只是线程间共享。

---

### 命名信号量

直接内核命名：

```cpp
sem_t* s = sem_open("/mysem", O_CREAT, 0644, 0);
```

这种天然适合跨进程：

* 不需要你自己把 `sem_t` 放共享内存
* 多个进程通过同一个名字打开即可

---

# 7. eventfd、pipe、socket 这些天然就是进程间的

这些你不用担心“能不能跨进程”，因为它们本来就是 IPC 机制。

例如：

* `pipe`
* `FIFO`
* `UNIX domain socket`
* `eventfd`

它们的特点是：

* 通知能力强
* 能阻塞等待
* 容易纳入 `poll/epoll`

所以工程里很常见：

* **共享内存存大数据**
* **eventfd/pipe/socket 做唤醒通知**

---

# 8. futex 也能跨进程

futex 本质上是对某个用户态内存地址上的整数做等待/唤醒。
只要这个整数在共享内存里，多个进程就能围绕它同步。

所以：

* 同进程线程可用
* 跨进程也可用

很多高性能跨进程锁/同步原语底层就是这么干的。

只是 futex 太底层，一般不手写，除非你在做基础库。

---

# 9. 一个最典型的跨进程条件变量模型

共享内存里：

```cpp
struct Shared {
    pthread_mutex_t mutex;
    pthread_cond_t cond;
    int ready;
    int value;
};
```

### 写进程

```cpp
pthread_mutex_lock(&s->mutex);
s->value = 42;
s->ready = 1;
pthread_cond_signal(&s->cond);
pthread_mutex_unlock(&s->mutex);
```

### 读进程

```cpp
pthread_mutex_lock(&s->mutex);
while (!s->ready) {
    pthread_cond_wait(&s->cond, &s->mutex);
}
int v = s->value;
pthread_mutex_unlock(&s->mutex);
```

这就是**跨进程**的，不是线程内专属。

因为：

* `s` 本身在共享内存里
* `mutex/cond` 也在共享内存里
* 它们的属性是 `PTHREAD_PROCESS_SHARED`

---

# 10. 为什么“锁/条件变量对象本身”也必须放共享内存

因为锁不是一个抽象概念，它本身就是一块状态数据。

比如一个 mutex 内部要记录：

* 是否加锁
* 谁持有
* 等待队列相关状态

如果进程 A 和进程 B 各自有一份 mutex 副本，那它们锁的就不是同一个东西，当然无法同步。

所以跨进程同步的本质是：

> **同步原语的状态也必须是共享的。**

---

# 11. 工程上怎么选

## 跨进程共享状态 + 等待唤醒

用：

* 共享内存 + `pthread_mutex_t` + `pthread_cond_t`

优点：

* 模型清晰
* 和线程版很像
* 适合“有数据就唤醒”

---

## 生产者消费者 / 环形队列

用：

* 共享内存 + semaphore
* 或共享内存 + mutex + cond

信号量在计数场景更顺手。

---

## 事件驱动服务器 / epoll 体系

用：

* 共享内存 + eventfd

这个很适合 Linux 工程。

---

## 极致性能、基础库开发

用：

* 共享内存 + atomic + futex

但更难写对。

---

# 12. 你可以这样记一句

**线程同步原语和进程同步原语不完全是两套东西；pthread 的 mutex/cond 在设置为 `PTHREAD_PROCESS_SHARED` 并放入共享内存后，也可以做跨进程同步。**

---

# 13. 最后帮你区分一下最容易混的两组

## 不能直接按“跨进程”理解的

```cpp
std::mutex
std::condition_variable
```

## 可以跨进程的

```cpp
pthread_mutex_t
pthread_cond_t
sem_t
eventfd
futex
```

但前两者要满足：

* 在共享内存中
* 属性设为 `PTHREAD_PROCESS_SHARED`
