Linux 下共享内存常用两套 API：

1. **System V**：`shmget / shmat / shmdt / shmctl`
2. **POSIX**：`shm_open / ftruncate / mmap / munmap / shm_unlink`

本质都一样：**让多个进程把同一块内存映射到自己地址空间里**。([man7.org][1])

---

## 一、System V 共享内存 API

### 1. `shmget`

作用：**创建或获取共享内存段**，返回 `shmid`。`shmget()` 可以获取已有共享内存，也可以在带 `IPC_CREAT` 时创建新段。([man7.org][1])

原型：

```cpp
int shmget(key_t key, size_t size, int shmflg);
```

常见写法：

```cpp
int shmid = shmget(key, sizeof(MyData), IPC_CREAT | 0644);
```

含义：

* `key`：共享内存的 key
* `size`：大小
* `IPC_CREAT`：不存在就创建
* `0644`：权限

只打开已有段：

```cpp
int shmid = shmget(key, 0, 0644);
```

这时如果不存在就失败。([man7.org][1])

---

### 2. `shmat`

作用：**把共享内存挂到当前进程地址空间**。`shmat()` 会把 `shmid` 对应的共享内存段附加到调用进程的地址空间；当 `shmaddr` 为 `NULL` 时，系统会选择一个合适的页对齐地址。([openSUSE手册页服务器][2])

原型：

```cpp
void* shmat(int shmid, const void* shmaddr, int shmflg);
```

常见写法：

```cpp
void* p = shmat(shmid, nullptr, 0);
if (p == (void*)-1) {
    perror("shmat");
}
```

然后转成你的结构体：

```cpp
MyData* data = static_cast<MyData*>(p);
```

只读挂载也可以：

```cpp
void* p = shmat(shmid, nullptr, SHM_RDONLY);
```

([openSUSE手册页服务器][2])

---

### 3. `shmdt`

作用：**解除当前进程的映射**。

```cpp
int shmdt(const void* shmaddr);
```

```cpp
shmdt(p);
```

它只是当前进程不再使用这块共享内存，不一定马上销毁整块共享内存。([openSUSE手册页服务器][2])

---

### 4. `shmctl`

作用：**控制共享内存**，比如删除。`shmctl()` 对 `shmid` 指定的共享内存段执行控制操作。([man7.org][3])

最常用是删除标记：

```cpp
shmctl(shmid, IPC_RMID, nullptr);
```

这表示把这段共享内存标记删除；真正何时消失取决于是否还有进程附加着它。Linux 还允许对已被 `IPC_RMID` 标记删除的段继续 `shmat()`，但这不是可移植行为。([man7.org][3])

## 二、System V 最小示例

### 写进程

```cpp
#include <sys/ipc.h>
#include <sys/shm.h>
#include <cstdio>
#include <cstring>

struct Data {
    int ready;
    int value;
};

int main() {
    key_t key = 0x12345;

    int shmid = shmget(key, sizeof(Data), IPC_CREAT | 0644);
    if (shmid == -1) {
        perror("shmget");
        return 1;
    }

    void* p = shmat(shmid, nullptr, 0);
    if (p == (void*)-1) {
        perror("shmat");
        return 1;
    }

    auto* data = static_cast<Data*>(p);
    data->value = 42;
    data->ready = 1;

    printf("writer done\n");

    shmdt(p);
    return 0;
}
```

### 读进程

```cpp
#include <sys/ipc.h>
#include <sys/shm.h>
#include <cstdio>

struct Data {
    int ready;
    int value;
};

int main() {
    key_t key = 0x12345;

    int shmid = shmget(key, 0, 0644);
    if (shmid == -1) {
        perror("shmget");
        return 1;
    }

    void* p = shmat(shmid, nullptr, SHM_RDONLY);
    if (p == (void*)-1) {
        perror("shmat");
        return 1;
    }

    auto* data = static_cast<const Data*>(p);
    printf("ready=%d value=%d\n", data->ready, data->value);

    shmdt(p);
    return 0;
}
```

## 三、POSIX 共享内存 API

现在也很常用，风格更像文件。

### 1. `shm_open`

作用：**创建或打开 POSIX 共享内存对象**。`shm_open()` 会建立到共享内存对象的连接，并返回一个文件描述符。([man7.org][4])

```cpp
int fd = shm_open("/myshm", O_CREAT | O_RDWR, 0644);
```

名字通常形如 `/myshm`。([Linux JM][5])

---

### 2. `ftruncate`

作用：**设置共享内存大小**。在 POSIX 共享内存流程里，通常 `shm_open()` 后要 `ftruncate()` 设定对象大小。([man7.org][6])

```cpp
ftruncate(fd, sizeof(MyData));
```

---

### 3. `mmap`

作用：**把这个共享内存对象映射到进程地址空间**。`mmap()` 成功返回映射地址，失败返回 `MAP_FAILED`（即 `(void*)-1`）。当 `addr == NULL` 时，内核会选择合适的页对齐地址。([man7.org][7])

```cpp
void* p = mmap(nullptr, sizeof(MyData),
               PROT_READ | PROT_WRITE,
               MAP_SHARED, fd, 0);
```

关键参数：

* `PROT_READ | PROT_WRITE`：可读可写
* `MAP_SHARED`：修改对其他映射该对象的进程可见。`MAP_SHARED` 的更新对其他映射同一区域的进程是可见的。([man7.org][7])

---

### 4. `munmap`

作用：解除映射。`munmap()` 解除先前建立的映射。([man7.org][7])

```cpp
munmap(p, sizeof(MyData));
```

---

### 5. `shm_unlink`

作用：删除共享内存对象。POSIX 共享内存对象会一直存在到系统关闭或被 `shm_unlink()` 删除。([Linux JM][5])

```cpp
shm_unlink("/myshm");
```

## 四、POSIX 最小示例

### 写进程

```cpp
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <cstdio>

struct Data {
    int ready;
    int value;
};

int main() {
    const char* name = "/myshm";

    int fd = shm_open(name, O_CREAT | O_RDWR, 0644);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    if (ftruncate(fd, sizeof(Data)) == -1) {
        perror("ftruncate");
        return 1;
    }

    void* p = mmap(nullptr, sizeof(Data),
                   PROT_READ | PROT_WRITE,
                   MAP_SHARED, fd, 0);
    if (p == MAP_FAILED) {
        perror("mmap");
        return 1;
    }

    auto* data = static_cast<Data*>(p);
    data->value = 100;
    data->ready = 1;

    munmap(p, sizeof(Data));
    close(fd);
    return 0;
}
```

### 读进程

```cpp
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <cstdio>

struct Data {
    int ready;
    int value;
};

int main() {
    const char* name = "/myshm";

    int fd = shm_open(name, O_RDONLY, 0644);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* p = mmap(nullptr, sizeof(Data),
                   PROT_READ,
                   MAP_SHARED, fd, 0);
    if (p == MAP_FAILED) {
        perror("mmap");
        return 1;
    }

    auto* data = static_cast<const Data*>(p);
    printf("ready=%d value=%d\n", data->ready, data->value);

    munmap(p, sizeof(Data));
    close(fd);
    return 0;
}
```

## 五、两套 API 怎么选

### System V

特点：

* 老牌
* 接口是 `key -> shmid -> shmat`
* 你前面代码就是这套

### POSIX

特点：

* 更像“文件 + mmap”
* 名字更直观
* 常和 `mmap`、`eventfd`、`sem_open` 之类一起用

如果是新项目，很多人更愿意用 **POSIX 共享内存**。POSIX 共享内存 API 的典型接口就是 `shm_open()`、`ftruncate()`、`mmap()`、`shm_unlink()`。([man7.org][6])

---

## 六、共享内存只负责“共享数据”，不负责“通知”

这个很重要。

你把数据写进共享内存之后，对方不会自动醒。通常还要配合：

* `pthread_mutex_t` / `pthread_cond_t`（进程共享）
* `sem_t`
* `eventfd`
* `pipe`
* `poll/epoll`

比如 `sem_init()` 里，`pshared != 0` 表示该信号量可在进程间共享，并且应放在共享内存区域里。([man7.org][8])

---

## 七、你先记这张流程图就够了

### System V

```text
key -> shmget -> shmid -> shmat -> 指针
```

### POSIX

```text
name -> shm_open -> fd -> ftruncate -> mmap -> 指针
```

---

## 八、一句话总结

共享内存 API 的核心用法就是：

* **创建/打开共享内存**
* **映射到本进程地址空间**
* **把返回地址当作结构体指针读写**
* **用完解除映射，必要时删除对象**。([man7.org][1])
