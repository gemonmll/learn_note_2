# 一、什么是 `std::atomic`？

`std::atomic` 是 C++11 提供的 **原子操作类型**，定义在：

```cpp
#include <atomic>
```

它的作用是：

> ✅ 让变量的读写在多线程下是原子的（不会被打断）
> ✅ 不需要加锁
> ✅ 避免数据竞争（data race）

---

# 二、为什么需要 atomic？

看一个经典错误例子：

```cpp
bool ready = false;

void worker() {
    ready = true;
}

void main_thread() {
    while (!ready) {
        // 等待
    }
}
```

❌ 问题：

* 多线程读写 `ready`
* 编译器可能优化
* CPU 可能缓存
* 可能永远读不到 true

这是 **未定义行为**

---

# 三、用 std::atomic<bool> 正确写法

```cpp
std::atomic<bool> ready(false);

void worker() {
    ready.store(true);
}

void main_thread() {
    while (!ready.load()) {
    }
}
```

✅ 现在是线程安全的

---

# 四、atomic 的基本用法

---

## 1️⃣ 定义

```cpp
std::atomic<bool> flag;
std::atomic<int> counter(0);
```

---

## 2️⃣ 读写

### 写

```cpp
flag.store(true);
```

### 读

```cpp
if (flag.load()) { }
```

---

## 3️⃣ 更简洁写法（推荐）

atomic 支持运算符重载：

```cpp
flag = true;        // 等价 store
bool x = flag;      // 等价 load
```

---

# 五、atomic<int> 常用操作

---

## 1️⃣ 自增（线程安全）

```cpp
std::atomic<int> count(0);

count++;
++count;
count.fetch_add(1);
```

这三种都是原子的。

---

## 2️⃣ compare_exchange（核心操作）

这是 atomic 最重要的能力。

```cpp
std::atomic<int> x(10);

int expected = 10;
x.compare_exchange_strong(expected, 20);
```

意思是：

> 如果 x == expected
> 就把 x 改成 20
> 否则更新 expected 为当前 x

这是实现无锁结构的核心。

---

# 六、std::atomic<bool> 常见用途

---

## 1️⃣ 线程退出控制

```cpp
std::atomic<bool> stop(false);

void worker() {
    while (!stop) {
        // 做事
    }
}

void stop_worker() {
    stop = true;
}
```

---

## 2️⃣ 代替 mutex 做简单标志位

```cpp
std::atomic<bool> initialized(false);

if (!initialized.exchange(true)) {
    // 只有一个线程会进来
}
```

`exchange` 是原子替换，返回旧值。

---

# 七、memory order（进阶）

默认情况下：

```cpp
flag.store(true);
```

使用的是：

```
memory_order_seq_cst
```

这是最安全的顺序一致性模型。

一般情况下：

> 🚀 你不需要管 memory order
> 默认就好

除非你在写 lock-free 容器。

---

# 八、atomic vs mutex 区别

| 对比 | atomic   | mutex |
| -- | -------- | ----- |
| 性能 | 快        | 慢     |
| 用途 | 简单变量     | 复杂逻辑  |
| 适合 | 计数器/flag | 多步骤操作 |

---

# 九、atomic 不能解决什么？

❌ 不能保证多个变量一起原子

错误例子：

```cpp
std::atomic<int> x, y;

x = 1;
y = 2;
```

这不是整体原子操作。

如果需要：

👉 用 mutex

---

# 十、和你前面 call_once 对比

| 场景       | 用什么          |
| -------- | ------------ |
| 只执行一次初始化 | call_once    |
| 线程退出标志   | atomic<bool> |
| 计数器      | atomic<int>  |
| 多步骤逻辑    | mutex        |

---

# 十一、典型面试级理解总结

> atomic 是基于 CPU 原子指令实现的无锁同步工具
> 它保证单个变量的操作不会被线程打断
> 但不保证复杂逻辑的原子性
