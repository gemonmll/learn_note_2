这是 **C++ 的类型别名（type alias）**，使用 `using` 关键字定义的。
它的作用和 `typedef` 一样，只是 **更现代、更清晰**。

我们逐行解释你的代码。

---

# 一、代码含义

```cpp
using Reader = BorrowedRawMessage;
using Builder = BorrowedRawMessage&;
```

等价于旧写法：

```cpp
typedef BorrowedRawMessage Reader;
typedef BorrowedRawMessage& Builder;
```

意思是：

| 别名        | 实际类型                  |
| --------- | --------------------- |
| `Reader`  | `BorrowedRawMessage`  |
| `Builder` | `BorrowedRawMessage&` |

---

# 二、第一行

```cpp
using Reader = BorrowedRawMessage;
```

表示：

```
Reader 就是 BorrowedRawMessage 的别名
```

使用时：

```cpp
Reader msg;
```

等价于：

```cpp
BorrowedRawMessage msg;
```

---

# 三、第二行（重点）

```cpp
using Builder = BorrowedRawMessage&;
```

这里定义的是：

```
Builder = BorrowedRawMessage 的引用类型
```

使用时：

```cpp
Builder b = msg;
```

等价于：

```cpp
BorrowedRawMessage& b = msg;
```

也就是说：

> Builder 不是对象
> 而是 **引用类型**

---

# 四、为什么要这么写？

这种写法在 **序列化 / RPC / Cap’n Proto / Protobuf** 代码里很常见。

通常含义是：

| 类型      | 作用   |
| ------- | ---- |
| Reader  | 只读访问 |
| Builder | 可写访问 |

例如：

```cpp
void process(Reader msg);   // 只读
void modify(Builder msg);   // 可修改
```

实际展开：

```cpp
void process(BorrowedRawMessage msg);
void modify(BorrowedRawMessage& msg);
```

---

# 五、为什么 Builder 用引用？

因为：

```
Builder = BorrowedRawMessage&
```

这样调用函数不会拷贝对象：

```cpp
void modify(Builder msg) {
    msg.set_value(10);
}
```

本质是：

```cpp
void modify(BorrowedRawMessage& msg)
```

修改会作用到原对象。

---

# 六、这种写法的设计模式

很多库会这样设计：

```cpp
class Message {
public:
    using Reader = Message;
    using Builder = Message&;
};
```

使用时：

```cpp
Message::Reader r;
Message::Builder b = r;
```

目的：

* API 语义清晰
* Reader / Builder 角色分离

---

# 七、一个完整示例

```cpp
class BorrowedRawMessage {
public:
    void set(int v) { value = v; }
    int get() const { return value; }

private:
    int value = 0;
};

using Reader = BorrowedRawMessage;
using Builder = BorrowedRawMessage&;

void print(Reader msg) {
    std::cout << msg.get() << std::endl;
}

void update(Builder msg) {
    msg.set(10);
}
```

---

# 八、现代 C++ 推荐用 using

旧写法：

```cpp
typedef std::vector<int> IntVec;
```

新写法：

```cpp
using IntVec = std::vector<int>;
```

原因：

* 更易读
* 支持模板 alias

例如：

```cpp
template<typename T>
using Vec = std::vector<T>;
```

`typedef` 做不到。

---

# 九、一句话总结

```cpp
using Reader = BorrowedRawMessage;
using Builder = BorrowedRawMessage&;
```

意思是：

```
Reader  = BorrowedRawMessage
Builder = BorrowedRawMessage 的引用
```

通常表示：

* **Reader：只读对象**
* **Builder：可修改对象**
