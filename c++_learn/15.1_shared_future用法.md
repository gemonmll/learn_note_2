`std::shared_future` 是 C++11 提供的 **可共享 future**。
它和 `std::future` 的最大区别是：

| 类型                   | get() 调用次数 | 是否可拷贝        | 典型用途       |
| -------------------- | ---------- | ------------ | ---------- |
| `std::future`        | **只能调用一次** | 不可拷贝，只能 move | 单消费者       |
| `std::shared_future` | **可以调用多次** | **可拷贝**      | 多线程等待同一个结果 |

因此 **`shared_future` 主要用于多个线程同时等待同一个异步结果**。

---

# 1 基本用法

最常见方式：
**先得到 `future` → 再 `.share()` 变成 `shared_future`**

```cpp
#include <iostream>
#include <future>
#include <thread>

int compute() {
    std::this_thread::sleep_for(std::chrono::seconds(1));
    return 42;
}

int main() {
    std::future<int> f = std::async(std::launch::async, compute);

    std::shared_future<int> sf = f.share();  // 转换为 shared_future

    std::thread t1([sf](){
        std::cout << "t1: " << sf.get() << std::endl;
    });

    std::thread t2([sf](){
        std::cout << "t2: " << sf.get() << std::endl;
    });

    t1.join();
    t2.join();
}
```

运行结果类似：

```
t1: 42
t2: 42
```

特点：

* `sf.get()` **可以调用多次**
* 多线程安全等待同一结果

---

# 2 shared_future 的核心特性

### 1 可复制

```cpp
std::shared_future<int> sf1 = f.share();
std::shared_future<int> sf2 = sf1;
std::shared_future<int> sf3 = sf1;
```

多个对象共享同一个状态。

---

### 2 get() 可以调用多次

```cpp
std::cout << sf.get() << std::endl;
std::cout << sf.get() << std::endl;
```

而 `future`：

```cpp
std::future<int> f = std::async(...);
f.get();   // OK
f.get();   // ❌ 抛异常
```

---

### 3 支持多个线程 wait()

```cpp
sf.wait();
sf.wait_for(std::chrono::seconds(1));
sf.wait_until(timepoint);
```

---

# 3 promise + shared_future

另一种常见模式：
**promise 生产数据，多个线程消费**

```cpp
#include <future>
#include <iostream>
#include <thread>

int main() {
    std::promise<int> p;

    std::shared_future<int> sf = p.get_future().share();

    std::thread t1([sf](){
        std::cout << "t1: " << sf.get() << std::endl;
    });

    std::thread t2([sf](){
        std::cout << "t2: " << sf.get() << std::endl;
    });

    std::this_thread::sleep_for(std::chrono::seconds(1));

    p.set_value(100);

    t1.join();
    t2.join();
}
```

输出

```
t1: 100
t2: 100
```

典型场景：

* 多个 worker 等待初始化完成
* 配置加载完成通知
* 一次计算结果多线程复用

# 4 shared_future 的实际使用场景

### 场景1：一次计算，多个线程读取

```cpp
std::shared_future<Model> model_future;
```

多个线程：

```
thread1 -> model_future.get()
thread2 -> model_future.get()
thread3 -> model_future.get()
```

只加载一次模型。

---

### 场景2：初始化同步

```cpp
std::shared_future<void> init_done;
```

所有线程：

```
init_done.wait();
```

---

### 场景3：线程池任务依赖

```
taskA -> produce result
taskB -> use result
taskC -> use result
```

用 `shared_future` 共享结果。

---

# 5 shared_future vs future 对比

| 特性       | future | shared_future |
| -------- | ------ | ------------- |
| get() 次数 | 1次     | 多次            |
| 是否可复制    | ❌      | ✅             |
| 多线程读取    | ❌      | ✅             |
| 内存开销     | 小      | 稍大            |
| 使用场景     | 单消费者   | 多消费者          |

---

# 6 常见坑

### 1 忘记 `.share()`

```cpp
auto f = std::async(...);

std::thread t1([f]() {  // ❌ future 不能 copy
});
```

必须：

```cpp
auto sf = f.share();
```

---

### 2 future 被 share 后失效

```cpp
auto f = std::async(...);

auto sf = f.share();

f.get();   // ❌ f 已经 empty
```

`share()` 会 **转移 state**。

---

# 7 shared_future 内部原理（简化）

```
shared_future
       │
       ▼
 shared state
       │
 ┌─────┴─────┐
thread1   thread2
```

shared state 存储：

* result
* exception
* ready flag
* condition_variable

所有 `shared_future` 共享这块状态。

---

# 8 一句话总结

**`std::shared_future` = 可复制 + 多线程共享结果的 future**

使用模式：

```
future → share() → shared_future
```

主要解决：

> **多个线程等待同一个异步结果**
