好。下面我给你补一个**跨进程生产者-消费者队列**的完整最小示例。

这个例子比前面的“单个 ready/value”更接近真实通信：

* 一个共享内存中的**环形队列**
* 一个**跨进程 mutex**
* 两个**跨进程条件变量**

  * `not_empty`：队列非空时通知消费者
  * `not_full`：队列非满时通知生产者

这样就不用轮询了，满了/空了都会阻塞等待。

---

# 一、整体思路

共享内存中放一个结构：

```text
[ mutex ]
[ not_empty ]
[ not_full ]
[ head ]
[ tail ]
[ count ]
[ buffer[N] ]
```

含义：

* `head`：读位置
* `tail`：写位置
* `count`：当前元素个数
* `buffer[N]`：环形缓冲区

规则：

### 生产者

* 如果队列满了：等 `not_full`
* 否则写入一个元素
* 写完通知 `not_empty`

### 消费者

* 如果队列空了：等 `not_empty`
* 否则取出一个元素
* 取完通知 `not_full`

# 二、完整代码

---

## 1）公共头文件 `queue_shared.h`

```cpp
#ifndef QUEUE_SHARED_H
#define QUEUE_SHARED_H

#include <pthread.h>

constexpr const char* kQueueShmName = "/my_ipc_queue_demo";
constexpr int kQueueSize = 8;

struct SharedQueue {
    pthread_mutex_t mutex;
    pthread_cond_t not_empty;
    pthread_cond_t not_full;

    int head;
    int tail;
    int count;

    int buffer[kQueueSize];
};

#endif
```

---

## 2）初始化程序 `queue_init.cpp`

```cpp
#include "queue_shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kQueueShmName, O_CREAT | O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    if (ftruncate(fd, sizeof(SharedQueue)) == -1) {
        perror("ftruncate");
        close(fd);
        shm_unlink(kQueueShmName);
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedQueue),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        shm_unlink(kQueueShmName);
        return 1;
    }

    auto* q = static_cast<SharedQueue*>(addr);

    pthread_mutexattr_t mutex_attr;
    pthread_condattr_t cond_attr;

    pthread_mutexattr_init(&mutex_attr);
    pthread_condattr_init(&cond_attr);

    pthread_mutexattr_setpshared(&mutex_attr, PTHREAD_PROCESS_SHARED);
    pthread_condattr_setpshared(&cond_attr, PTHREAD_PROCESS_SHARED);

    if (pthread_mutex_init(&q->mutex, &mutex_attr) != 0) {
        fprintf(stderr, "pthread_mutex_init failed\n");
        return 1;
    }

    if (pthread_cond_init(&q->not_empty, &cond_attr) != 0) {
        fprintf(stderr, "pthread_cond_init not_empty failed\n");
        return 1;
    }

    if (pthread_cond_init(&q->not_full, &cond_attr) != 0) {
        fprintf(stderr, "pthread_cond_init not_full failed\n");
        return 1;
    }

    q->head = 0;
    q->tail = 0;
    q->count = 0;

    pthread_mutexattr_destroy(&mutex_attr);
    pthread_condattr_destroy(&cond_attr);

    munmap(addr, sizeof(SharedQueue));
    close(fd);

    printf("queue init done\n");
    return 0;
}
```

## 3）生产者 `producer.cpp`

```cpp
#include "queue_shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kQueueShmName, O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedQueue),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    auto* q = static_cast<SharedQueue*>(addr);

    for (int x = 1; x <= 20; ++x) {
        pthread_mutex_lock(&q->mutex);

        while (q->count == kQueueSize) {
            printf("producer: queue full, wait...\n");
            pthread_cond_wait(&q->not_full, &q->mutex);
        }

        q->buffer[q->tail] = x;
        q->tail = (q->tail + 1) % kQueueSize;
        q->count++;

        printf("producer: push %d, count=%d\n", x, q->count);

        pthread_cond_signal(&q->not_empty);
        pthread_mutex_unlock(&q->mutex);

        usleep(200000);  // 模拟生产速度
    }

    munmap(addr, sizeof(SharedQueue));
    close(fd);
    return 0;
}
```

## 4）消费者 `consumer.cpp`

```cpp
#include "queue_shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kQueueShmName, O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedQueue),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    auto* q = static_cast<SharedQueue*>(addr);

    for (int i = 0; i < 20; ++i) {
        pthread_mutex_lock(&q->mutex);

        while (q->count == 0) {
            printf("consumer: queue empty, wait...\n");
            pthread_cond_wait(&q->not_empty, &q->mutex);
        }

        int v = q->buffer[q->head];
        q->head = (q->head + 1) % kQueueSize;
        q->count--;

        printf("consumer: pop %d, count=%d\n", v, q->count);

        pthread_cond_signal(&q->not_full);
        pthread_mutex_unlock(&q->mutex);

        usleep(350000);  // 模拟消费速度
    }

    munmap(addr, sizeof(SharedQueue));
    close(fd);
    return 0;
}
```

## 5）清理程序 `queue_cleanup.cpp`

```cpp
#include "queue_shared.h"

#include <fcntl.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <unistd.h>

#include <cstdio>
#include <cstdlib>

int main() {
    int fd = shm_open(kQueueShmName, O_RDWR, 0666);
    if (fd == -1) {
        perror("shm_open");
        return 1;
    }

    void* addr = mmap(nullptr, sizeof(SharedQueue),
                      PROT_READ | PROT_WRITE,
                      MAP_SHARED, fd, 0);
    if (addr == MAP_FAILED) {
        perror("mmap");
        close(fd);
        return 1;
    }

    auto* q = static_cast<SharedQueue*>(addr);

    pthread_mutex_destroy(&q->mutex);
    pthread_cond_destroy(&q->not_empty);
    pthread_cond_destroy(&q->not_full);

    munmap(addr, sizeof(SharedQueue));
    close(fd);

    shm_unlink(kQueueShmName);

    printf("queue cleanup done\n");
    return 0;
}
```

# 五、你要重点看懂的地方

## 1. 为什么要两个条件变量

因为这里有两个不同的“等待条件”：

* 生产者等：`队列非满`
* 消费者等：`队列非空`

所以最好拆成两个条件变量：

* `not_full`
* `not_empty`

这样逻辑清晰，唤醒也更精准。

---

## 2. 为什么都要 `while`

生产者：

```cpp
while (q->count == kQueueSize) {
    pthread_cond_wait(&q->not_full, &q->mutex);
}
```

消费者：

```cpp
while (q->count == 0) {
    pthread_cond_wait(&q->not_empty, &q->mutex);
}
```

因为条件变量只表示“你可以再检查一次条件了”，不保证条件绝对成立。

---

## 3. 为什么 `signal` 放在加锁区域里

这版写法里：

```cpp
pthread_cond_signal(&q->not_empty);
pthread_mutex_unlock(&q->mutex);
```

这样做是很常见也很稳妥的，因为：

* 状态更新和通知属于同一个临界区逻辑
* 被唤醒方拿到锁后，一定能看到一致状态

---

## 4. 这个模型已经是“跨进程通信”了

虽然代码长得很像多线程版生产者消费者，但它其实是：

* 不同进程
* 共享内存中的同一个 mutex
* 共享内存中的同一个 cond
* 共享内存中的同一个队列状态

这就是跨进程版。

# 六、这个例子和你前面学的内容怎么串起来

你前面问过：

* 共享内存是不是只能一直查
* 条件变量能不能跨进程
* 锁能不能跨进程

这个例子把答案全连起来了：

1. **共享内存**负责存队列和同步原语
2. **跨进程 mutex** 保证对队列状态修改互斥
3. **跨进程 cond** 让满/空时阻塞等待，不用轮询

所以完整通信不是“只靠共享内存”，而是：

**共享内存 + 共享锁 + 共享条件变量**

# 七、再给你一个脑中时序图

### 生产者快、消费者慢时

* 生产者不断 push
* 队列满了
* 生产者阻塞在 `not_full`
* 消费者 pop 一个后 `signal(not_full)`
* 生产者被唤醒继续写

### 消费者快、生产者慢时

* 消费者不断 pop
* 队列空了
* 消费者阻塞在 `not_empty`
* 生产者 push 一个后 `signal(not_empty)`
* 消费者被唤醒继续读
