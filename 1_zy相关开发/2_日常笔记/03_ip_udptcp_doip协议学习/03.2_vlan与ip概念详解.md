## 疑问 vlan会改ip地址吗，比如172.20.0.68经过vlan2 会变成172.20.2.68吗
不会。**VLAN 不会自动改 IP 地址。**

`172.20.0.68` 经过 VLAN 2，不会因为 VLAN ID 是 2 就变成 `172.20.2.68`。

VLAN 是二层概念，它改的是**以太网帧头里面的 VLAN Tag**，不是三层 IP 头。

---

## 1. VLAN 改的是哪里？

普通以太网帧大概是：

```text
Dst MAC | Src MAC | EtherType=IPv4 | IP Header | TCP/UDP | Data
```

带 VLAN 的以太网帧是：

```text
Dst MAC | Src MAC | VLAN Tag | EtherType=IPv4 | IP Header | TCP/UDP | Data
```

VLAN Tag 里有 VLAN ID，比如：

```text
VLAN ID = 2
```

但是 IP 头里面仍然是：

```text
Src IP = 172.20.0.68
Dst IP = 172.20.0.100
```

所以它不会变成：

```text
172.20.2.68
```

---

## 2. 那为什么经常看到 VLAN 2 对应 172.20.2.x？

这是人为规划，不是 VLAN 自动转换。

工程里经常这么设计：

```text
VLAN 1  -> 172.20.1.0/24
VLAN 2  -> 172.20.2.0/24
VLAN 10 -> 172.20.10.0/24
VLAN 20 -> 172.20.20.0/24
```

这样做是为了好记：

```text
VLAN ID = 2
IP 网段 = 172.20.2.0/24
```

但这只是网络规划习惯，不是协议强制规定。

你完全可以这样配：

```text
VLAN 2 -> 192.168.100.0/24
VLAN 2 -> 10.10.10.0/24
VLAN 2 -> 172.20.0.0/24
```

协议上都可以。

---

## 3. VLAN 2 里面可以用 172.20.0.68 吗？

可以，只要同一个 VLAN 内的设备 IP 规划一致。

例如：

```text
PC-A:
VLAN 2
IP = 172.20.0.68/24

PC-B:
VLAN 2
IP = 172.20.0.100/24
```

它们在同一个 VLAN 2，并且 IP 也在同一个网段 `172.20.0.0/24`，就可以直接通信。

---

## 4. 不同 VLAN 之间能不能用同一个 IP 网段？

理论上可以，但工程上一般不建议。

例如：

```text
VLAN 2:
PC-A = 172.20.0.68/24

VLAN 3:
PC-B = 172.20.0.100/24
```

虽然 IP 看起来同网段，但它们处于不同 VLAN，二层广播隔离。

PC-A 想找 `172.20.0.100` 时，会认为它在本地直连网段，于是发 ARP：

```text
Who has 172.20.0.100?
```

但是 ARP 广播只在 VLAN 2 内传播，进不了 VLAN 3。

所以 PC-B 收不到，PC-A 也就通信不了。

这就是为什么工程上通常会规划成：

```text
一个 VLAN 对应一个独立 IP 子网
```

例如：

```text
VLAN 2 -> 172.20.2.0/24
VLAN 3 -> 172.20.3.0/24
```

---

## 5. VLAN 和 IP 的关系

可以这样理解：

```text
VLAN：二层隔离域
IP 网段：三层地址规划
```

推荐做法是：

```text
一个 VLAN 对应一个 IP 子网
```

但不是协议强制关系。

比如：

```text
VLAN 2 不一定必须是 172.20.2.0/24
VLAN 10 不一定必须是 172.20.10.0/24
```

只是人为约定。

---

## 6. Linux 上的例子

假设物理网卡是 `eth0`，创建 VLAN 2 子接口：

```bash
sudo ip link add link eth0 name eth0.2 type vlan id 2
sudo ip addr add 172.20.0.68/24 dev eth0.2
sudo ip link set eth0.2 up
```

这里的含义是：

```text
eth0.2：表示走 VLAN ID 2
172.20.0.68/24：表示这个 VLAN 接口上的 IP 地址
```

不是说 `.2` 会自动把 IP 变成 `172.20.2.68`。

你也可以配成：

```bash
sudo ip addr add 192.168.50.68/24 dev eth0.2
```

照样可以，只要对端也在 VLAN 2 且 IP 网段匹配。

---

## 7. 一句话总结

```text
VLAN ID 不会改 IP。
VLAN 2 只是二层标签，表示这个以太网帧属于 VLAN 2。
172.20.2.x 只是工程上常见的 IP 规划习惯，不是 VLAN 自动转换结果。
```

所以你的例子：

```text
172.20.0.68 经过 VLAN 2
```

实际更准确地说是：

```text
IP 仍然是 172.20.0.68
以太网帧外面多了一个 VLAN ID = 2 的二层标签
```
