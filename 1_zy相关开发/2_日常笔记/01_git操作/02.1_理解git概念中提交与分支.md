## 正常链路指向 head->分支 ->提交
HEAD -> main -> C
区别很大，但可以用一句话先抓住：

> **提交（commit）是“历史快照节点”，分支（branch）是“指向某个提交的可移动名字”。**

---

## 1. 提交是什么

提交就是你一次 `git commit` 产生的一个**历史记录点**。

它里面通常包含：

* 当前文件树的快照
* 提交说明
* 作者
* 时间
* 它的父提交是谁

可以把它想成链表里的一个节点：

```text
A <- B <- C
```

这里：

* `A` 是一个旧提交
* `B` 基于 `A`
* `C` 基于 `B`

每个提交一旦生成，**内容基本就固定了**。
也就是说，提交本身通常**不会动**。

---

## 2. 分支是什么

分支本质上只是一个名字，指向某个提交。

比如：

```text
main -> C
```

表示分支 `main` 现在指向提交 `C`。

所以分支不是一堆代码，不是提交集合本体，而是：

> **“当前这条开发线最新到哪里了”的一个指针**

---

## 3. 最核心的区别

### 提交

* 是一个具体历史节点
* 记录一次版本状态
* 生成后基本不变
* 有自己的 SHA 哈希

### 分支

* 只是一个引用名
* 指向某个提交
* 会随着新提交不断移动
* 用来表示一条开发线当前的位置

---

## 4. 举个最直观的例子

假设现在有：

```text
A <- B <- C
         ^
       main
```

意思是：

* 有 3 个提交：A、B、C
* `main` 分支当前指向 `C`

这时你再提交一次，生成 `D`：

```text
A <- B <- C <- D
              ^
            main
```

你会发现：

* **提交 `C` 没变**
* **提交 `D` 是新建出来的**
* **分支 `main` 从 `C` 挪到了 `D`**

所以：

* 提交像一个个车站
* 分支像“当前列车停在哪个站”的标记牌

---

## 5. 为什么容易混

因为平时你说“main 分支的代码”，其实真正代码状态来自 **main 当前指向的那个提交**。

也就是说：

* 分支自己不存代码内容
* 真正的内容在提交里
* 分支只是告诉你“该看哪个提交”

---

## 6. 从 HEAD 角度再看一遍

正常情况下：

```text
HEAD -> main -> C
```

意思是：

* 你当前在 `main` 分支上
* `main` 指向提交 `C`

当你 commit 一次：

* Git 新建一个提交 `D`
* 然后把 `main` 挪到 `D`

变成：

```text
HEAD -> main -> D
```

所以你可以看出：

* **提交是新建**
* **分支是移动**

这就是它们最本质的差别。

---

## 7. 再用一个类比

可以把 Git 想成一本书：

### 提交

像书里的某一页快照
比如第 10 页、第 20 页、第 30 页

### 分支

像一个书签
书签插在哪一页，就表示这条线当前看到哪儿了

所以：

* 页本身不会因为你挪书签而改变
* 书签可以随时移动
* 你新写一页后，书签会移到新页上

---

## 8. 命令上怎么体现

看提交历史：

```bash
git log --oneline
```

你看到的是提交。

看分支：

```bash
git branch
```

你看到的是分支名。

看当前分支指向哪里：

```bash
git log --oneline --decorate
```

可能看到：

```text
abc1234 (HEAD -> main) fix bug
def5678 add feature
```

这表示：

* `abc1234` 是一个提交
* `main` 这个分支当前指向它

---

## 9. 一句话彻底记住

可以这样背：

* **commit = 版本点 / 快照点**
* **branch = 指向某个版本点的可移动指针**

或者更形象一点：

* **提交是点**
* **分支是指针**
* **HEAD 是当前你站在哪个分支上的指针**

---

## 10. 最后给你一个最小图

```text
A <- B <- C
         ^
       main
```

* `A/B/C` 是提交
* `main` 是分支
* 分支只是指着某个提交

如果你再提交一次：

```text
A <- B <- C <- D
              ^
            main
```

* `D` 是新提交
* `main` 移动到 `D`

这张图基本就把“提交和分支的区别”说完了。
