## merge是新建一个提交，git add 后 git commit
## rebase是要基于旧的生成，git add 后 git rebase --continue 

`git merge` 遇到冲突时，思路比 `rebase` 更直一点：

**先合并，冲突就停下，你手动改，`git add`，最后自己 `git commit`。**

---

## 1. merge 冲突时发生了什么

比如你在自己分支上执行：

```bash
git checkout my-branch
git merge main
```

Git 会尝试把 `main` 合进 `my-branch`。

如果有冲突，它会停下来，提示类似：

```bash
Auto-merging foo.c
CONFLICT (content): Merge conflict in foo.c
Automatic merge failed; fix conflicts and then commit the result.
```

意思是：

* Git 试图自动合并
* 某些文件合不上
* 现在需要你手动处理
* 处理完后，由你来提交这次 merge

---

## 2. 冲突文件里是什么样

冲突文件里通常会出现这样的标记：

```text
<<<<<<< HEAD
这是你当前分支 my-branch 的内容
=======
这是 main 分支过来的内容
>>>>>>> main
```

你要手动编辑成你真正想要的最终结果。

例如改成：

```text
这是合并后的最终内容
```

并把那些 `<<<<<<<`、`=======`、`>>>>>>>` 标记删掉。

---

## 3. 解决冲突的标准步骤

假设冲突发生后：

### 第一步：看状态

```bash
git status
```

你会看到哪些文件 unmerged / both modified。

---

### 第二步：手动改文件

打开冲突文件，按你的意图改好。

---

### 第三步：标记已解决

改完后：

```bash
git add <文件名>
```

如果多个文件都改完了：

```bash
git add .
```

这一步表示：

> 这些冲突我已经处理好了，暂存的是最终合并结果。

---

### 第四步：提交 merge

当所有冲突都解决并 add 后，执行：

```bash
git commit
```

Git 会生成一个 merge commit。

很多时候会自动弹出默认提交信息，像：

```text
Merge branch 'main' into my-branch
```

你可以直接保存退出，也可以改成更清楚一点的说明。

---

## 4. 和 rebase 最大区别

### merge 冲突后

你是这样收尾：

```bash
git add ...
git commit
```

### rebase 冲突后

你是这样收尾：

```bash
git add ...
git rebase --continue
```

因为：

* `merge` 是在创建一个新的 **merge commit**
* `rebase` 是在继续“重放旧 commit”

---

## 5. 一个完整例子

```bash
git checkout my-branch
git merge main
# 出现冲突

git status
# 手动编辑冲突文件

git add foo.c
git add bar.c
git commit
```

提交完后，merge 就完成了。

然后再 push：

```bash
git push origin my-branch
```

注意这里通常**不需要**像 rebase 那样 `--force-with-lease`，因为 merge 不会改写已有历史。

---

## 6. 如果想放弃这次 merge

如果你发现冲突太乱，不想继续了：

```bash
git merge --abort
```

这会回到 merge 开始前的状态。

---

## 7. 一个很实用的理解

你可以把 merge 想成：

> “把 main 的内容拿过来，和我当前分支揉在一起，揉不动的地方我手工处理，然后我自己做一个合并提交。”

所以 merge 的冲突解决流程就是：

```bash
git merge main
# 冲突
git status
# 手改文件
git add ...
git commit
git push
```

---

## 8. 推荐你记住这两个模板

### merge 冲突模板

```bash
git merge main
git status
# 手动解决冲突
git add <file>
git commit
git push origin my-branch
```

### rebase 冲突模板

```bash
git rebase main
git status
# 手动解决冲突
git add <file>
git rebase --continue
git push --force-with-lease origin my-branch
```

---

## 9. 一句话总结

**`git merge` 冲突后：手动改文件 → `git add` → `git commit`。**
和 `rebase` 不同，`merge` 最后是你自己提交一个 merge commit。
