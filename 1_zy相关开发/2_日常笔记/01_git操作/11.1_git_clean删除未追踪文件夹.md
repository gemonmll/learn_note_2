`git clean -fd` 用来**删除 Git 没有跟踪的文件/目录**。

拆开看：

```bash
git clean -fd
```

* `-f`：force，强制执行。Git 默认不允许直接 clean，必须加 `-f` 才会真的删。
* `-d`：directories，也删除未跟踪的目录。

所以：

```bash
git reset --hard HEAD
git clean -fd
```

通常表示：**把工作区彻底恢复到当前提交的干净状态**。

区别是：

```bash
git reset --hard HEAD
```

会丢弃**已被 Git 跟踪的文件**的修改，比如你改了已有文件。

```bash
git clean -fd
```

会删除**未被 Git 跟踪的文件和文件夹**，比如新建但还没 `git add` 的文件、编译产物、临时目录等。

举例：

```text
tracked.txt      # Git 已跟踪，你修改了
new.txt          # 新建文件，Git 未跟踪
dist/            # 新建目录，Git 未跟踪
```

执行后：

```bash
git reset --hard HEAD
```

会还原 `tracked.txt`。

```bash
git clean -fd
```

会删除 `new.txt` 和 `dist/`。

注意：这个删除通常**不可恢复**。执行前可以先预览：

```bash
git clean -fdn
```

其中 `-n` 是 dry run，只显示将要删除什么，不真的删除。
