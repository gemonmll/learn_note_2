1) git reset 移动head指针，且恢复快照（配合git clean -fd来删除未追踪的文件夹）

普通恢复
git fetch origin
git reset --hard origin/develop

彻底恢复
git reset --hard HEAD 
git clean -fd


2）git restore 不移动指针，恢复快照 

注意恢复暂存区和工作区
git fetch origin
git restore --source=origin/develop --staged --worktree .

3）git rebase 压缩本体提交，修改本体提交 

4) git rebase 合并主分支

5）git merge 合并主分支

6）git fetch 只更新部合并，注意与git pull区别

合入远端代码
git fetch origin
git merge origin/develop
但如果要恢复成远端分支，最好使用reset (合入不等于恢复)
git fetch origin
git reset --hard origin/develop


7）git commit --amend 指令，合并提交

8）git reset + cherrypick 来恢复本地冲突文件

9）git reset hard 与soft 恢复

10）使用插件git history 来比较版本的差异（git merge只能比较合入与自己分支的差异，无法比较合入和云端差异）
多使用compare tree和beyong compare来比较不同版本差异

11)git log 和git ref =log

12）git lfs取消追踪
```bash
git lfs untrack "路径/文件名"
git add .gitattributes
git rm --cached "路径/文件名"
git add "路径/文件名"
git commit -m "stop tracking file with git lfs"
cicd push（卓驭限定）
```
13） git lfs 追踪
```bash
git lfs untrack "*.zip"
git add .gitattributes
git commit -m "Stop tracking zip files with Git LFS"
```
