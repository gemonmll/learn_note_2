正常模式下、**非哨兵模式**，一般不应该请求哨兵 PNC，也就是：

```text
PNC8 = 0
```

但注意：**不是说 HAD 完全不发 NM**，而是看 HAD 当前是否还需要参与正常网络管理。

---

## 1. 正常行车 / 上电模式

如果车辆处于正常上电、行车、诊断、智驾可用等状态，HAD 可能仍然会发自己的 NM，比如：

```text
HAD Tx 0x598 NM
```

但里面的哨兵相关 PNC 位应该是：

```text
PNC8 = 0
```

含义是：

```text
HAD 现在不请求哨兵网络
```

如果此时 HAD 需要其他功能网络，可能会置其他 PNC 位，比如 PNC1、PNC2、PNC3 等，具体看你们网管矩阵定义。

所以非哨兵正常模式可能是：

```text
0x598 NM：
PNC8 = 0
其他 PNC = 根据当前功能请求决定
```

---

## 2. 正常模式下不需要 HAD 保持网络时

如果车辆准备下电，且 HAD 没有业务需要保持唤醒，那么 HAD 应该释放自己的网络请求。

可能表现为：

```text
HAD 停发 0x598
```

或者：

```text
HAD 继续短时间发 NM，但 PNC8 = 0，其他 PNC 也为 0
```

具体是“停发”还是“发全 0”，要看你们平台的 NM 状态机和网络管理规范。

但本质是一样的：

```text
HAD 不再请求任何 PNC 保持
```

---

## 3. 和哨兵模式的区别

你可以这样对比：

| 场景           | HAD 0x598 是否发 |                PNC8 |
| ------------ | ------------: | ------------------: |
| 正常行车 / 上电    |           通常发 |                   0 |
| 正常行车但有其他网络请求 |             发 | PNC8=0，其他 PNC 按需为 1 |
| 哨兵模式开启       |             发 |                   1 |
| 哨兵重启前释放网络    |    停发或 PNC8=0 |                     |
| 重启后等待 IA     |          暂不外发 |                     |
| IA 确认恢复哨兵    |             发 |                   1 |
| 完全休眠         |            不发 |                     |

---

## 4. 放到 0x598 和 0x5C1 里看

非哨兵正常模式下，如果 HAD 发：

```text
HAD → 0x598：PNC8 = 0
```

PDC-Master 汇总后，如果没有其他节点请求 PNC8，它就应该发：

```text
PDC-Master → 0x5C1：PNC8 = 0
```

表示：

```text
哨兵相关网络没有被请求，可以不用保持唤醒
```

如果车辆处于正常上电状态，其他网络可能仍然是醒的，但不是因为 PNC8 醒，而是因为：

```text
KL15 / ignition / 用户上电
普通 NM 保活
其他 PNC 请求
诊断会话
应用通信需求
```

---

一句话说：

**非哨兵正常模式下，HAD 可以正常发 0x598 NM，但哨兵对应的 PNC8 应该为 0；只有进入哨兵模式或需要哨兵相关网络保持时，才把 PNC8 置 1。**
