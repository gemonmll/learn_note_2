帮我学习下打桩机制，我看有一些c文件通过注册函数，就可以在shell脚本中输入指令执行函数，这是怎么做的

你说的这种“在 C 文件里注册一个函数，然后在 shell 里输入命令就能执行”，通常不是 shell 直接认识了你的 C 函数，而是**程序内部做了一层命令注册表**。
本质上是这几步：

1. 先定义一种“命令 -> 函数”的映射关系
2. 各个 c 文件把自己的命令注册进去
3. 程序启动时把这些命令收集起来
4. shell 输入一行字符串后，程序解析命令名
5. 在注册表里找到对应函数并调用

这就是很多嵌入式系统、诊断工具、调试 CLI、UT 打桩框架里常见的机制。

---

## 一、先把现象翻译成底层逻辑

比如你在 shell 里输入：

```sh
dump_can 1
```

看起来像 shell 在执行 `dump_can`，但其实常见情况是：

* 你启动了一个可执行程序，比如 `diag_cli`
* 它内部有个循环：

  * 读一行输入
  * 解析成 `cmd = "dump_can"`，参数是 `"1"`
  * 在命令表里查找 `dump_can`
  * 找到后调用对应的 C 函数

也就是：

```c
dump_can_handler(argc, argv);
```

所以真正执行函数的是**你的 CLI 框架**，不是 Linux shell 自己直接调用 C 函数。

---

## 二、最核心的数据结构

通常会有一个命令描述结构体：

```c
typedef int (*cmd_func_t)(int argc, char *argv[]);

typedef struct {
    const char *name;
    const char *help;
    cmd_func_t func;
} cli_cmd_t;
```

这里表示一条命令有：

* `name`：命令名
* `help`：帮助信息
* `func`：真正执行的函数指针

比如某个模块里写：

```c
static int cmd_dump_can(int argc, char *argv[])
{
    printf("dump can called\n");
    return 0;
}
```

再把它注册成一条命令：

```c
static cli_cmd_t g_cmd = {
    .name = "dump_can",
    .help = "dump can messages",
    .func = cmd_dump_can
};
```

---

# 三、注册通常有哪几种实现方式

最常见有 3 类。

---

## 方式 1：显式调用注册函数

这是最容易理解的。

### 1）提供全局注册接口

```c
#define MAX_CMD_NUM 128

static cli_cmd_t *g_cmd_table[MAX_CMD_NUM];
static int g_cmd_num = 0;

void cli_register(cli_cmd_t *cmd)
{
    if (g_cmd_num < MAX_CMD_NUM) {
        g_cmd_table[g_cmd_num++] = cmd;
    }
}
```

### 2）各模块主动注册

```c
static int cmd_dump_can(int argc, char *argv[])
{
    printf("dump can called\n");
    return 0;
}

static cli_cmd_t g_dump_can_cmd = {
    .name = "dump_can",
    .help = "dump can messages",
    .func = cmd_dump_can
};

void can_cli_init(void)
{
    cli_register(&g_dump_can_cmd);
}
```

### 3）系统启动时统一调用

```c
void cli_init(void)
{
    can_cli_init();
    dem_cli_init();
    dcm_cli_init();
}
```

### 4）命令执行

```c
int cli_execute(const char *input)
{
    char cmd_name[64];
    // 这里只演示，真正代码会做 argv 拆分
    sscanf(input, "%63s", cmd_name);

    for (int i = 0; i < g_cmd_num; i++) {
        if (strcmp(g_cmd_table[i]->name, cmd_name) == 0) {
            return g_cmd_table[i]->func(0, NULL);
        }
    }

    printf("unknown command: %s\n", cmd_name);
    return -1;
}
```

### 优点

* 简单直观
* 容易调试

### 缺点

* 每加一个模块，都得记得在初始化里手动调一次
* 容易漏注册

---

## 方式 2：用宏 + 特定段（linker section）自动收集

这个就是你看到“有些 c 文件只写个注册宏，没看到谁调 init，但命令就生效了”的常见原因。

思路是：

* 每个 c 文件定义一个 `cli_cmd_t`
* 并把它放进一个特殊的链接段，比如 `.cli_cmd`
* 启动时只要知道这段的起止地址，就能把所有命令遍历出来

---

### 例子

#### 1）定义命令结构体

```c
typedef int (*cmd_func_t)(int argc, char *argv[]);

typedef struct {
    const char *name;
    const char *help;
    cmd_func_t func;
} cli_cmd_t;
```

#### 2）定义注册宏

GCC 常见写法：

```c
#define CLI_CMD_EXPORT(_name, _help, _func) \
    const cli_cmd_t __cli_cmd_##_func       \
    __attribute__((used, section("cli_cmd"))) = { \
        _name, _help, _func \
    }
```

#### 3）模块里直接写

```c
static int cmd_dump_can(int argc, char *argv[])
{
    printf("dump can called\n");
    return 0;
}

CLI_CMD_EXPORT("dump_can", "dump can messages", cmd_dump_can);
```

这时候这个 `__cli_cmd_cmd_dump_can` 变量会被编译器放到 `cli_cmd` 段里。

#### 4）主程序遍历段

链接脚本里一般会提供段的起止符号，例如：

```c
extern const cli_cmd_t __start_cli_cmd[];
extern const cli_cmd_t __stop_cli_cmd[];
```
