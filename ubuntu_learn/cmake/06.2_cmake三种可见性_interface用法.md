# 🎯 CMake 的 include 有三种可见性

```
PRIVATE
PUBLIC
INTERFACE
```

你可以这样理解：

| 关键字           | 谁能看到 include？ | 本 target 本身编译时用吗？ | 下游 target 编译时用吗？ |
| ------------- | ------------- | ----------------- | ---------------- |
| **PRIVATE**   | 只有自己          | ✔ 用               | ✖ 不给别人           |
| **PUBLIC**    | 自己 + 所有链接到它的人 | ✔ 用               | ✔ 传播给别人          |
| **INTERFACE** | 只给别人          | ✖ 自己不用            | ✔ 给别人用           |

---

# 🚀 **重点解释 INTERFACE 的意思**

## **INTERFACE = “我不需要，但别人需要我提供”**

换句话说：

* 这个 target **自己不需要这个 include 目录进行编译**
* 但是 **依赖它的下游 target 需要这个 include 目录编译**
* 所以这个 include 目录 **只传递给下游，不作用于本库**

### 场景例子理解（最容易懂）：

你有一个**头文件-only** 的库：

```
myheaderlib/
 └── include/
       └── MyHeader.hpp
```

你创建这个库：

```cmake
add_library(myheaderlib INTERFACE)
target_include_directories(myheaderlib INTERFACE include)
```

解释：

* myheaderlib 不需要编译（因为它没有 cpp 文件）
  → 所以它“不需要 include 目录”
* 但是别的库/程序要使用它的头文件
  → 所以必须告诉 CMake “用户可以 include 这个目录”

因此：

```
INTERFACE = 只给使用者，不给自己
```

# 🔨 再举一个简单例子（你一看就懂）

## 示例 1：PRIVATE

```cmake
target_include_directories(zltoolkit PRIVATE src)
```

意思：

* zltoolkit 库 **需要用 src 做编译**
* 但别的 target **不继承 src**

---

## 示例 2：PUBLIC

```cmake
target_include_directories(zltoolkit PUBLIC src)
```

意思：

* zltoolkit **用 src 编译**
* 任何链接 zltoolkit 的程序（例如 ADAS_Replay_Tool）也自动有 `src` include

---

## 示例 3：INTERFACE（你问的重点）

```cmake
target_include_directories(zltoolkit INTERFACE src)
```

意思：

* zltoolkit **自己不需要 src 做编译**
  （通常用于 header-only 库）
* 链接 zltoolkit 的程序会自动 **继承 src**

# 🧠 再简单一点的人话解释

### PRIVATE

👉 “我自己用，不给别人”

### PUBLIC

👉 “我自己要用，别人也要用”

### INTERFACE

👉 “我自己不用，别人需要我提供”
