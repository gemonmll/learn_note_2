因为你这条命令：

```bash
git checkout -b feature/avm_pcsim_appNode
```

**只是“从当前分支的当前提交（HEAD）新建一个本地分支”**，它**不会**自动去把远端的 `origin/feature/avm_pcsim_appNode` 分支内容切过来。

你现在 `ls` 只看到 `README.md`，说明你“创建分支时所在的那个分支/提交”本来就只有 README（很常见：`master` 是空骨架，真正代码在 `develop` 或某个 feature 分支）。

---

## 你现在要切到远端真正的 feature 分支内容

先看看远端有哪些分支（确认远端分支确实存在）：

```bash
git branch -a
```

你现在本地已经有了同名分支 `feature/avm_pcsim_appNode`（但内容是从当前 HEAD 来的），所以最简单是：**删掉它，再按远端分支重新建**：

```bash
# 1) 切回一个安全分支（比如 master）
git switch master

# 2) 删除你刚刚建错的本地分支
git branch -D feature/avm_pcsim_appNode

# 3) 从远端分支创建并跟踪
git switch -c feature/avm_pcsim_appNode --track origin/feature/avm_pcsim_appNode
```

切完再看：

```bash
ls
git status
```

这时你应该就能看到那堆目录文件了。

---

## 如果你的代码其实在 develop（也很常见）

直接切到 develop：

```bash
git switch -c develop --track origin/develop
```

---

## 为什么一定要这样做（一句话）

* `git checkout -b XXX`：从**当前提交**复制一份分支指针（不去远端拿内容）
* `git switch -c XXX --track origin/XXX`：创建本地分支并绑定到**远端同名分支**，内容就是远端那条分支的代码

如果你把 `git branch -a` 的输出贴出来，我可以告诉你：代码到底在 `origin/develop` 还是在 `origin/feature/avm_pcsim_appNode`。
