舍弃“当前分支的 commit”通常有 3 种需求：**本地还没 push**、**已经 push**、**只想丢掉某几个 commit**。按你情况选一种就行。

---

## 1）最后一个 commit 想直接丢掉（还没 push）

### 丢掉 commit + 丢掉改动（最干净）

```bash
git reset --hard HEAD~1
```

### 丢掉 commit，但保留改动在工作区（等于“撤回提交继续改”）

```bash
git reset --soft HEAD~1
```

### 丢掉 commit，改动放到暂存区（介于 soft 和 hard）

```bash
git reset --mixed HEAD~1
```

---

## 2）丢掉最近 N 个 commit（还没 push）

```bash
git reset --hard HEAD~N
```

例如丢掉最近 3 个：

```bash
git reset --hard HEAD~3
```

---

## 3）commit 已经 push 到远端了（推荐用 revert，不改历史）

撤销某个 commit（生成一个新 commit 来回退它）：

```bash
git revert <commit_hash>
git push
```

如果要撤销一段连续 commit：

```bash
git revert <oldest_hash>^..<newest_hash>
git push
```

---

## 4）如果你就是想“远端也一起丢掉”（强制改历史，谨慎）

```bash
git reset --hard HEAD~1
git push -f origin <branch_name>
```

多人协作分支别轻易用 `-f`，容易把别人提交冲掉。
